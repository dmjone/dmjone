Act I · The Dispatch Floor

A 112 dispatcher's day, at this moment.

An emergency call arrives. The clock starts. Every second between "hello" and "unit en route" is paid in someone else's life.

Before · Status quo

Four screens. Three radios. Two databases.

  • CAD console for incident logging
  • GIS map for unit locations
  • Roster spreadsheet for who's on shift
  • Phone for callers, radio for units
  • Memory for "what did the supervisor decide last hour"
  • Stress for the rest
After · CityGuard

One screen. One feed. One AI tactical plan.

  • Live map renders incidents in priority order
  • Keyboard arrows step through the queue
  • Enter selects an incident, Enter again asks Gemini
  • Sub-second AI plan with units, route and protocol
  • Firebase persists every decision with audit context
  • One pair of hands, full situational awareness

Act II · The Four Pillars

Four Google services, one operational reality.

No mocks. No fakes. Each service does exactly one job, and every job is load-bearing.

Pillar 01 · Map

Google Maps

Live incident visualisation with interactive markers. Clusters, priority colours, haversine distance maths, object pooling for high-frequency map updates.

@vis.gl/react-google-maps

Pillar 02 · AI

Gemini 2.0 Flash

Tactical response plan in sub-second. Units to dispatch, route to take, protocol to follow, hazards to flag. Streamed direct to the AI command panel.

@google/genai

Pillar 03 · Auth + DB

Firebase

Google sign-in for dispatchers, Firestore for persistent incident state. Every decision carries the human and the time, ready for after-action review.

firebase / Auth + Firestore

Pillar 04 · Telemetry

Google Analytics 4

Usage tracking, performance metrics, accessibility insights. Web Vitals (LCP, FID, CLS, TTFB, FCP) measured in production, not just CI.

gtag.js · dynamic load

Act V · Proof

Production-grade from day one.

220 tests · zero mocking

Real Google service integrations exercised in CI: workflow tests for dispatch, integration tests for module interactions, 46 accessibility tests alone.

Responsive: phone, tablet, desktop, 4K

Mobile uses bottom-tab navigation with safe-area support for notched devices. Tablet shifts to two-column. Desktop opens the full three-column command-centre layout.

Object pooling for map performance

High-frequency marker updates reuse pooled objects to prevent GC stutter. Pool statistics tracked. Memory monitored with custom timing marks.

Hackathon brief: 100% on six parameters

Real-Time Resource Management Hub challenge. Built to prove the brief, not to demo. Strict TypeScript, no any, no mocks, no shortcuts.
