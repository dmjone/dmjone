2. Graphics Hardware & Display Technologies

Graphics hardware forms the physical interface between computational data and human visual perception. Its role is to capture human intent (input), convert abstract data into electrical signals, store visual states, and finally emit light or marks that the human eye can interpret as images.

2.1 Input Devices

Input devices convert human actions into digital signals. In graphics, the emphasis is on spatial accuracy, responsiveness, and continuous interaction rather than textual data alone.

2.1.1 Mouse

A mouse is a relative positioning device that reports displacement rather than absolute position.

  • Core idea: Movement on a surface → change in screen coordinates.
  • Why useful: Enables continuous 2D navigation, selection, dragging, and drawing.
  • Graphics relevance: Ideal for pointing, object manipulation, viewport control.
2.1.2 Keyboard

A keyboard is a discrete input device that generates symbolic commands.

  • Core idea: Key press → encoded character or control signal.
  • Graphics relevance: Mode switching, parameter entry, shortcuts, command-driven graphics.
2.1.3 Light Pen

A light pen detects light emitted from the screen to determine the point being touched.

  • Core idea: Synchronization with CRT beam position.
  • Key property: Absolute pointing device.
  • Limitation: Works reliably only with CRT displays.
2.1.4 Graphic Tablet

A graphic tablet (digitizer) maps stylus position directly to screen coordinates.

  • Core idea: Absolute coordinate mapping.
  • Graphics relevance: Natural drawing, handwriting, pressure-sensitive input.
  • Use case: CAD, digital art, animation keyframe creation.
2.1.5 Scanner

A scanner converts physical images into pixel data.

  • Core idea: Sampling reflected light into a raster image.
  • Graphics relevance: Texture acquisition, document digitization, image processing.

2.2 Output Devices

Output devices transform digital pixel data into visible or physical representations.

2.2.1 CRT (Cathode Ray Tube)

CRT displays use an electron beam to excite phosphor-coated screens.

  • Raster scanning: Electron beam sweeps the screen line by line.
  • Phosphor persistence: Duration for which phosphor continues glowing after excitation.

Persistence controls flicker and motion blur. High persistence reduces flicker but increases smearing; low persistence requires higher refresh rates.

2.2.2 LCD / LED Monitors

Flat-panel displays where pixels are fixed physical elements.

  • Core idea: Individual pixels modulate light.
  • Advantage: No beam scanning, low power, stable image.
  • Graphics implication: Resolution is native and fixed.
2.2.3 Plotters

Plotters produce vector drawings using pens.

  • Core idea: Draws continuous lines rather than pixels.
  • Use case: Engineering drawings, architectural plans.
2.2.4 Printers

Printers generate raster or vector output on physical media.

  • Graphics relevance: Hard-copy output of rendered images.
  • Resolution: Measured in DPI (dots per inch).

2.3 Display Techniques

Display techniques define how image data is refreshed and drawn on the screen.

2.3.1 Raster Scan Display

Raster displays refresh the entire screen sequentially.

  • Scanning pattern: Left-to-right, top-to-bottom (zig-zag).
  • Concept: Every pixel is refreshed, regardless of change.
  • Used in: CRT, LCD, LED monitors.
2.3.2 Random Scan Display

Also known as vector scan displays.

  • Core idea: Draws only the required lines directly.
  • Advantage: High-quality lines.
  • Limitation: Poor at filled areas and complex scenes.
2.3.3 DVST (Direct View Storage Tube)

A special CRT that stores the image permanently until erased.

  • Core idea: No refresh required.
  • Limitation: No selective erase, no animation.

2.4 Display Parameters & Definitions

Display parameters quantify visual quality, timing, and geometric correctness.

2.4.1 Resolution

Number of pixels on the screen.

Definition: Horizontal × Vertical pixels.

2.4.2 Aspect Ratio

Ratio of width to height.

$$\text{Aspect Ratio} = \frac{\text{Width}}{\text{Height}}$$

2.4.3 Refresh Rate

Number of times the screen is refreshed per second.

  • Unit: Hertz (Hz)
  • Higher value: Smoother motion, less flicker.
2.4.4 Frame Time

Time taken to display one frame.

$$\text{Frame Time} = \frac{1}{\text{Refresh Rate}}$$

2.4.5 Dot Clock Frequency

Rate at which pixels are processed.

  • Unit: MHz
  • Controls: Maximum achievable resolution and refresh rate.
2.4.6 Interlacing

Technique where alternate scan lines are refreshed in successive passes.

  • Purpose: Reduce bandwidth.
  • Drawback: Flicker and motion artifacts.

2.5 Frame Buffer

The frame buffer is the core memory structure of raster graphics systems.

2.5.1 Definition

A dedicated memory that stores pixel intensity or color values before display.

2.5.2 Bit Depth

Number of bits used to represent a single pixel.

  • Higher bit depth: More colors.
  • Example: 24-bit → 8 bits each for R, G, B.
2.5.3 Memory Calculation Formula

Total frame buffer memory required:

$$\text{Memory} = \text{Resolution} \times \text{Bit Depth}$$

This formula explains why higher resolution and richer color demand more memory and bandwidth.
