2. Graphics Hardware & Display Technologies

Graphics hardware forms the physical interface between computational data and human visual perception. Its role is to capture human intent (input), convert abstract data into electrical signals, store visual states, and finally emit light or marks that the human eye can interpret as images.

The Bridge

Q: What is the primary role of graphics hardware?

A: To act as the physical interface converting abstract data into light/images for human perception.

2.1 Input Devices

Input devices convert human actions into digital signals. In graphics, the emphasis is on spatial accuracy, responsiveness, and continuous interaction rather than textual data alone.

2.1.1 Mouse

A mouse is a relative positioning device that reports displacement rather than absolute position.

  • Core idea: Movement on a surface → change in screen coordinates.
  • Why useful: Enables continuous 2D navigation, selection, dragging, and drawing.
  • Graphics relevance: Ideal for pointing, object manipulation, viewport control.
Input Logic

Q: Is a mouse an absolute or relative positioning device?

A: Relative. It tracks displacement (movement), not grid position.

2.1.2 Keyboard

A keyboard is a discrete input device that generates symbolic commands.

  • Core idea: Key press → encoded character or control signal.
  • Graphics relevance: Mode switching, parameter entry, shortcuts, command-driven graphics.
2.1.3 Light Pen

A light pen detects light emitted from the screen to determine the point being touched.

  • Core idea: Synchronization with CRT beam position.
  • Key property: Absolute pointing device.
  • Limitation: Works reliably only with CRT displays.
Legacy Tech

Q: Why do light pens require a CRT monitor?

A: They rely on detecting the electron beam's scan timing to calculate position.

2.1.4 Graphic Tablet

A graphic tablet (digitizer) maps stylus position directly to screen coordinates.

  • Core idea: Absolute coordinate mapping.
  • Graphics relevance: Natural drawing, handwriting, pressure-sensitive input.
  • Use case: CAD, digital art, animation keyframe creation.
Precise Input

Q: Why are Graphic Tablets preferred for CAD and Art?

A: They provide Absolute Mapping (1:1 with screen) and pressure sensitivity.

2.1.5 Scanner

A scanner converts physical images into pixel data.

  • Core idea: Sampling reflected light into a raster image.
  • Graphics relevance: Texture acquisition, document digitization, image processing.

2.2 Output Devices

Output devices transform digital pixel data into visible or physical representations.

2.2.1 CRT (Cathode Ray Tube)

CRT displays use an electron beam to excite phosphor-coated screens.

  • Raster scanning: Electron beam sweeps the screen line by line.
  • Phosphor persistence: Duration for which phosphor continues glowing after excitation.

Persistence controls flicker and motion blur. High persistence reduces flicker but increases smearing; low persistence requires higher refresh rates.

CRT Physics

Q: What is "Persistence" in a CRT?

A: The duration the phosphor coating continues to glow after being struck by the electron beam.

2.2.2 LCD / LED Monitors

Flat-panel displays where pixels are fixed physical elements.

  • Core idea: Individual pixels modulate light.
  • Advantage: No beam scanning, low power, stable image.
  • Graphics implication: Resolution is native and fixed.
LCD vs CRT

Q: Why don't LCDs flicker like CRTs?

A: Pixels are fixed physical elements that stay "on" or "off" until changed; there is no scanning beam decay.

2.2.3 Plotters

Plotters produce vector drawings using pens.

  • Core idea: Draws continuous lines rather than pixels.
  • Use case: Engineering drawings, architectural plans.
2.2.4 Printers

Printers generate raster or vector output on physical media.

  • Graphics relevance: Hard-copy output of rendered images.
  • Resolution: Measured in DPI (dots per inch).

2.3 Display Techniques

Display techniques define how image data is refreshed and drawn on the screen.

2.3.1 Raster Scan Display

Raster displays refresh the entire screen sequentially.

  • Scanning pattern: Left-to-right, top-to-bottom (zig-zag).
  • Concept: Every pixel is refreshed, regardless of change.
  • Used in: CRT, LCD, LED monitors.
Raster Scan

Q: Describe the scanning pattern of a Raster Display.

A: Zig-zag: Left-to-right, then flyback, then next line down.

2.3.2 Random Scan Display

Also known as vector scan displays.

  • Core idea: Draws only the required lines directly.
  • Advantage: High-quality lines.
  • Limitation: Poor at filled areas and complex scenes.
Vector Scan

Q: What is the main advantage of Random (Vector) Scan displays?

A: Perfectly smooth lines (no "jaggies" or aliasing) because the beam follows the line path directly.

2.3.3 DVST (Direct View Storage Tube)

A special CRT that stores the image permanently until erased.

  • Core idea: No refresh required.
  • Limitation: No selective erase, no animation.

2.4 Display Parameters & Definitions

Display parameters quantify visual quality, timing, and geometric correctness.

2.4.1 Resolution

Number of pixels on the screen.

Definition: Horizontal × Vertical pixels.

Resolution

Q: How is Resolution defined?

A: Horizontal Pixels × Vertical Pixels (e.g., 1920 × 1080).

2.4.2 Aspect Ratio

Ratio of width to height.

$$\text{Aspect Ratio} = \frac{\text{Width}}{\text{Height}}$$

2.4.3 Refresh Rate

Number of times the screen is refreshed per second.

  • Unit: Hertz (Hz)
  • Higher value: Smoother motion, less flicker.
Refresh

Q: What is the unit of Refresh Rate?

A: Hertz (Hz) (times per second).

2.4.4 Frame Time

Time taken to display one frame.

$$\text{Frame Time} = \frac{1}{\text{Refresh Rate}}$$

2.4.5 Dot Clock Frequency

Rate at which pixels are processed.

  • Unit: MHz
  • Controls: Maximum achievable resolution and refresh rate.
Limits

Q: What does Dot Clock Frequency limit?

A: The maximum resolution and refresh rate a display interface can handle.

Time & Frequency Calculations

Timing and frequency calculations in raster graphics are derived from the refresh rate, screen resolution, and data transfer characteristics. These determine how fast pixels are scanned, transmitted, and displayed.

Example 1: Time Available to Scan One Horizontal Line

Given:

  • Resolution: 512 × 512 pixels
  • Refresh rate: 30 Hz
  • Horizontal retrace time: 4.28 μs

Step 1: Time for one full frame

$$T_{frame} = \frac{1}{\text{Refresh Rate}} = \frac{1}{30} = 0.03333 \text{ s}$$

Step 2: Total number of horizontal lines

$$N_{lines} = 512$$

Step 3: Time per horizontal line (including retrace)

$$T_{line} = \frac{T_{frame}}{N_{lines}} = \frac{0.03333}{512} \approx 6.51 \times 10^{-5} \text{ s}$$

$$T_{line} \approx 65.1 \text{ μs}$$

Step 4: Active scan time per line

$$T_{scan} = T_{line} - T_{retrace}$$

$$T_{scan} = 65.1 - 4.28 = 60.82 \text{ μs}$$

Time available to scan one horizontal line: ≈ 60.8 μs

Scan Time

Q: Why must we subtract retrace time?

A: During retrace (beam moving back), no pixels are drawn. Only the remaining time is "active".

Example 2: Dot Clock Frequency Calculation

Definition: Dot Clock Frequency is the rate at which individual pixels (dots) are transmitted to the display. It represents pixels drawn per second.

Given:

  • Resolution: 512 × 512 pixels
  • Refresh rate: 30 Hz

Step 1: Total pixels per frame

$$512 \times 512 = 262{,}144 \text{ pixels}$$

Step 2: Pixels displayed per second

$$\text{Pixels/sec} = 262{,}144 \times 30 = 7{,}864{,}320$$

Step 3: Dot Clock Frequency

$$f_{dot} = 7{,}864{,}320 \text{ Hz}$$

Dot Clock Frequency: ≈ 7.86 MHz

Example 3: Time to Load a Frame Buffer

Given:

  • Resolution: 640 × 480 pixels
  • Color depth: 12 bits per pixel
  • Data transmission rate: $10^5$ bits/second

Step 1: Total number of pixels

$$640 \times 480 = 307{,}200 \text{ pixels}$$

Step 2: Total data in bits

$$307{,}200 \times 12 = 3{,}686{,}400 \text{ bits}$$

Step 3: Time required to load frame buffer

$$T = \frac{\text{Total bits}}{\text{Transmission rate}}$$

$$T = \frac{3{,}686{,}400}{10^5} = 36.864 \text{ s}$$

Time required to load frame buffer: ≈ 36.86 seconds

2.4.6 Interlacing

Technique where alternate scan lines are refreshed in successive passes.

  • Purpose: Reduce bandwidth.
  • Drawback: Flicker and motion artifacts.
Interlacing

Q: What is the trade-off of Interlacing?

A: It halves the bandwidth needed but introduces visual artifacts like combing or flicker.

2.5 Frame Buffer

The frame buffer is the core memory structure of raster graphics systems.

It stores pixel intensity or color values before display.

Frame Buffer Memory Calculation

The frame buffer stores the color information of every pixel displayed on the screen. Memory requirement depends on three parameters: horizontal resolution, vertical resolution, and bits per pixel (color depth).

General formula:

$$\text{Frame Buffer Size (bits)} = \text{Width} \times \text{Height} \times \text{Bits per Pixel}$$

To convert bits to bytes: $$1 \text{ byte} = 8 \text{ bits}$$

To convert bytes to megabytes (MB): $$1 \text{ MB} = 1024 \times 1024 \text{ bytes}$$

Memory Logic

Q: What is the Frame Buffer formula?

A: \(\text{Memory} = \text{Width} \times \text{Height} \times \text{Bits-per-Pixel}\).

Example 1: 640 × 480 Resolution with 12 Bits per Pixel

Given:

  • Resolution: 640 × 480 pixels
  • Color depth: 12 bits per pixel

Step 1: Total number of pixels

$$640 \times 480 = 307{,}200 \text{ pixels}$$

Step 2: Total memory in bits

$$307{,}200 \times 12 = 3{,}686{,}400 \text{ bits}$$

Step 3: Convert bits to bytes

$$\frac{3{,}686{,}400}{8} = 460{,}800 \text{ bytes}$$

Memory required: 460,800 bytes (≈ 450 KB)

Example 2: 1024 × 768 Resolution with True Color (24-bit)

Given:

  • Resolution: 1024 × 768 pixels
  • Color depth: 24 bits per pixel

Step 1: Total number of pixels

$$1024 \times 768 = 786{,}432 \text{ pixels}$$

Step 2: Total memory in bits

$$786{,}432 \times 24 = 18{,}874{,}368 \text{ bits}$$

Step 3: Convert bits to bytes

$$\frac{18{,}874{,}368}{8} = 2{,}359{,}296 \text{ bytes}$$

Step 4: Convert bytes to MB

$$\frac{2{,}359{,}296}{1024 \times 1024} \approx 2.25 \text{ MB}$$

Frame buffer size: ≈ 2.25 MB

Example 3: 1200 × 1080 Resolution with 16-bit Color Depth

Given:

  • Resolution: 1200 × 1080 pixels
  • Color depth: 16 bits per pixel

Step 1: Total number of pixels

$$1200 \times 1080 = 1{,}296{,}000 \text{ pixels}$$

Step 2: Total memory in bits

$$1{,}296{,}000 \times 16 = 20{,}736{,}000 \text{ bits}$$

Step 3: Convert bits to bytes

$$\frac{20{,}736{,}000}{8} = 2{,}592{,}000 \text{ bytes}$$

Memory required: 2,592,000 bytes (≈ 2.47 MB)

2.5.1 Definition

A dedicated memory that stores pixel intensity or color values before display.

2.5.2 Bit Depth

Number of bits used to represent a single pixel.

  • Higher bit depth: More colors.
  • Example: 24-bit → 8 bits each for R, G, B.
Color Depth

Q: How many bits are needed for True Color (RGB)?

A: 24 bits (8 Red + 8 Green + 8 Blue).

2.5.3 Memory Calculation Formula

Total frame buffer memory required:

$$\text{Memory} = \text{Resolution} \times \text{Bit Depth}$$

This formula explains why higher resolution and richer color demand more memory and bandwidth.

Hardware Essentials

  • Input Devices: Convert human actions to digital signals (Mouse, Light Pen, Digitizer).
  • CRT (Cathode Ray Tube): Uses electron beams on phosphor screens; governed by persistence (glow duration) and refresh rate.
  • Raster Scan: Refreshes entire screen line-by-line (zig-zag pattern); good for realistic images but lower resolution than vector.
  • Random Scan (Vector): Draws lines directly; distinct for sharp lines (CAD) but poor for filled/shaded areas.
  • Frame Buffer Formula: \(\text{Memory per Frame} = \text{Resolution} \times \text{Bit Depth}\).
  • Dot Clock Frequency: Rate at which pixels are pushed to display; \(f_{dot} = H_{res} \times V_{res} \times \text{Refresh Rate}\).
  • Interlacing: Refreshing odd/even lines alternately to reduce bandwidth but may cause flicker.
