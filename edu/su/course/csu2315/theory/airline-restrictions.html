1. Problem Statement & Logic

Problem: Chef has 3 bags with weights $A$, $B$, and $C$. He needs to check in exactly two bags and carry one with him. The airline has strict rules:

  • The sum of the weights of the two checked-in bags must be $\le D$.
  • The weight of the bag he carries must be $\le E$.
Determine if Chef can travel with all three bags.

Logic: We simply need to test all three possible combinations of carrying one bag and checking the other two.

  • Option 1: Check-in $A$ and $B$, carry $C$. Valid if $(A+B \le D)$ AND $(C \le E)$.
  • Option 2: Check-in $A$ and $C$, carry $B$. Valid if $(A+C \le D)$ AND $(B \le E)$.
  • Option 3: Check-in $B$ and $C$, carry $A$. Valid if $(B+C \le D)$ AND $(A \le E)$.

If any of these conditions is met, the answer is "YES". Otherwise, "NO".

1.1 Dry Run

Example: $A=1, B=1, C=1, D=15, E=5$

  • Check Option 1: Carry $C$ ($1 \le 5$?), Check $A+B$ ($1+1=2 \le 15$?). Both true.
  • Result: Since one option worked, output YES.

Example: $A=10, B=20, C=30, D=20, E=40$

  • Option 1 (Carry 30, Check 10+20=30): $30 \le 40$ (True), but $30 \le 20$ (False).
  • Option 2 (Carry 20, Check 10+30=40): $20 \le 40$ (True), but $40 \le 20$ (False).
  • Option 3 (Carry 10, Check 20+30=50): $10 \le 40$ (True), but $50 \le 20$ (False).
  • Result: All fail. Output NO.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ A, B, C, D, E
    
    matchFound = FALSE
    
    # Check Case 1: Carry C
    IF (A + B <= D) AND (C <= E):
        matchFound = TRUE
    
    # Check Case 2: Carry B
    ELSE IF (A + C <= D) AND (B <= E):
        matchFound = TRUE
        
    # Check Case 3: Carry A
    ELSE IF (B + C <= D) AND (A <= E):
        matchFound = TRUE
        
    IF matchFound:
        PRINT "YES"
    ELSE:
        PRINT "NO"
  • Time Complexity: $O(1)$ per test case (constant number of checks).
  • Space Complexity: $O(1)$.
Why do we need to check 3 different cases?

A: Because any one of the three bags ($A, B,$ or $C$) could be the one Chef carries suitable for satisfying the constraint. We must verify if any valid configuration exists.
