Discrepancies in the Voters List

View Problem Online

1. Problem Statement & Logic

Problem: Given three lists of voter IDs ($N1, N2, N3$ integers respectively). Find the list of voters who appear in at least two of the three lists. Output the count and the sorted IDs.

Logic:

  • We can combine all three lists into one large list.
  • Then count the frequency of each ID.
  • If frequency of an ID $\ge 2$, include it in the final result.
  • Sort the final result and print.
  • Implementation detail: Use a Map/Frequency Array to count, or Sort the combined list and iterate to count consecutive duplicates. Sorting combined list $O((N1+N2+N3)\log \text{Total})$.

1.1 Dry Run

Example: L1=[1, 2], L2=[2, 3], L3=[1, 3]

  • Combined: [1, 2, 2, 3, 1, 3]
  • Frequencies: 1$\to$2, 2$\to$2, 3$\to$2.
  • All appear $\ge 2$ times.
  • Result: 1, 2, 3. Count: 3.

1.2 Pseudocode Solution


READ N1, N2, N3
READ List1, List2, List3

Combined = List1 + List2 + List3
SORT Combined

FinalList = []
i = 0
WHILE i < Length(Combined):
    count = 1
    current = Combined[i]
    while (i+1 < Length) AND (Combined[i+1] == current):
        count = count + 1
        i = i + 1
    
    IF count >= 2:
        APPEND current to FinalList
    i = i + 1

PRINT Length(FinalList)
FOR id in FinalList:
    PRINT id
  • Time Complexity: $O(N \log N)$ due to sorting.
  • Space Complexity: $O(N)$ for storage.
What is the most efficient way to solve this without a Hash Map?

A: Combine all lists into one, sort it ($O(N \log N)$), and then iterate to count consecutive occurrences. This avoids the space overhead/complexity of a hash map implementation in languages like C.
