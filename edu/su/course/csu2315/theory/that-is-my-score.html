1. Problem Statement & Logic

Problem: Contest has 11 problems (1-11). Only 1-8 are scorable. Calculate total score as sum of max score obtained for each problem 1-8.

Logic:

  • Initialize an array/map to store max score for problems 1-8 (default 0).
  • Iterate through submissions.
  • If problem index $p \le 8$:
    • Update `max_scores[p] = max(max_scores[p], score)`.
  • Sum up values in `max_scores` array.

1.1 Dry Run

Example: Submissions: (2, 40), (2, 50), (9, 100)

  • (2, 40): Prob 2 max = 40.
  • (2, 50): Prob 2 max = 50.
  • (9, 100): Prob 9 > 8. Ignore.
  • Total = 50.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    
    max_scores = [0] * 9  # Index 1 to 8
    
    FOR j FROM 1 TO N:
        READ P, S
        IF P <= 8:
            IF S > max_scores[P]:
                max_scores[P] = S
                
    total = 0
    FOR k FROM 1 TO 8:
        total = total + max_scores[k]
        
    PRINT total
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$ (Fixed size array).
Why do we strictly filter for $P \le 8$?

A: The problem specifies that only the first 8 problems (indices 1-8) are scorable for the total competition score. Problems 9, 10, and 11 do not contribute and must be ignored.
