Fit in Data Type

1. Problem Statement & Logic

Problem: A custom data type can store values from $0$ to $N$. If a value $X > N$ is assigned, it wraps around (cyclically). Find the stored value.

Logic:

  • The range size is actually $N+1$ values ($0, 1, \dots, N$).
  • This is equivalent to the modulo operation, but we need to represent it carefully.
  • Specifically, the problem says $N+1$ becomes 0, $N+2$ becomes 1. This means the value stored is $X \pmod{(N+1)}$.
  • Check if this formula holds: $X=N \implies N \pmod{N+1} = N$. Correct.
  • $X=N+1 \implies (N+1) \pmod{N+1} = 0$. Correct.
  • So the answer is $X \% (N+1)$.

1.1 Dry Run

Example: $N=10, X=12$

  • Range is $0 \dots 10$ (11 values).
  • $12 \% 11 = 1$.
  • Manual check: 0..10 is normal. 11$\to$0, 12$\to$1. Correct. Output: 1.

Example: $N=5, X=3$

  • $3 \% 6 = 3$. Output: 3.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N, X
    
    ans = X % (N + 1)
    PRINT ans
  • Time Complexity: $O(1)$.
  • Space Complexity: $O(1)$.
Why is the divisor $N+1$ and not $N$?

A: The data type holds values from $0$ to $N$, which is a total of $N+1$ distinct states. Wrapping occurs after reaching largest value $N$ (i.e., at $N+1$), so the modulus base is the total number of states ($N+1$).
