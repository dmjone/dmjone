Laptop Recommendation

View Problem Online

1. Problem Statement & Logic

Problem: Chef asks $N$ friends for laptop recommendations (values 1-10). He buys the one with maximum recommendations. If there's a tie for maximum, he is "CONFUSED".

Logic:

  • Input array $A$ of recommendations.
  • Count frequency of each laptop ID ($1$ to $10$).
  • Find the maximum frequency `max_freq`.
  • Count how many laptop IDs have frequency equal to `max_freq`.
  • If count is 1, print that ID.
  • If count > 1, print "CONFUSED".

1.1 Dry Run

Example: $A = [4, 4, 4, 3, 3]$

  • Freq: 3$\to$2, 4$\to$3. Others$\to$0.
  • Max Freq = 3.
  • IDs with freq 3: [4]. Count = 1.
  • Output: 4.

Example: $A = [4, 4, 3, 3]$

  • Freq: 4$\to$2, 3$\to$2.
  • Max Freq = 2.
  • IDs with freq 2: [3, 4]. Count = 2.
  • Output: CONFUSED.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ A (array)
    
    Freq = [0] * 11  # Indices 1 to 10
    
    FOR val in A:
        Freq[val] = Freq[val] + 1
        
    max_freq = 0
    FOR j FROM 1 TO 10:
        IF Freq[j] > max_freq:
            max_freq = Freq[j]
            
    winner = -1
    count_winners = 0
    FOR j FROM 1 TO 10:
        IF Freq[j] == max_freq:
            winner = j
            count_winners = count_winners + 1
            
    IF count_winners == 1:
        PRINT winner
    ELSE:
        PRINT "CONFUSED"
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$ (freq array size 11).
Why do we need a second pass to count winners?

A: We first determine the maximum frequency in Pass 1. Only then can we count exactly how many items achieved that specific maximum frequency in Pass 2 to detect a tie ("CONFUSED").
