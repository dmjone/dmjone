1. Problem Statement & Logic

Problem: Ashish has $X$ rupees. Pizza costs $Y$, Burger costs $Z$. He wants one item. Preference: Pizza > Burger > Nothing. Find what he buys.

Logic:

  • Check if he can afford Pizza ($X \ge Y$).
    • If yes, he buys "PIZZA". Stop.
  • If no ($X < Y$), check if he can afford Burger ($X \ge Z$).
    • If yes, he buys "BURGER". Stop.
  • If neither ($X < Y$ AND $X < Z$), he buys "NOTHING" .

1.1 Dry Run

Example: $X=50, Y=40, Z=20$

  • Can buy Pizza ($50 \ge 40$)? Yes. Output: PIZZA.

Example: $X=30, Y=40, Z=20$

  • Can buy Pizza ($30 \ge 40$)? No.
  • Can buy Burger ($30 \ge 20$)? Yes. Output: BURGER.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ X, Y, Z
    
    IF X >= Y:
        PRINT "PIZZA"
    ELSE IF X >= Z:
        PRINT "BURGER"
    ELSE:
        PRINT "NOTHING"
  • Time Complexity: $O(1)$.
  • Space Complexity: $O(1)$.
Why is the order of checks important (Pizza first)?

A: The problem states a strict preference: Pizza > Burger. We must check if Pizza is affordable first. Only if it is NOT affordable do we consider the Burger. Reversing the order would violate the preference.
