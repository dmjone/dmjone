1. Problem Statement & Logic

Problem: Given a binary string $S$ of length $N$, find the length of the longest alternating substring you can form by rearranging the characters of $S$. An alternating string is one where no two adjacent characters are the same (e.g., "0101" is alternating, "001" is not).

Logic: Since we can rearrange the characters arbitrarily, we just need to count the number of 0s and 1s.

  • Let $C_0$ be the count of '0's and $C_1$ be the count of '1's.
  • To make an alternating string, we strictly alternate 0s and 1s.
  • We can match every '0' with a '1' until one type runs out. This forms pairs of length $2 \cdot \min(C_0, C_1)$.
  • If we have any characters left over (Case $C_0 \neq C_1$), we can append exactly one of the remaining characters to the end. We cannot add more because that would create adjacent identical characters.

Formula: Answer = $2 \cdot \min(C_0, C_1) + (1 \text{ if } C_0 \neq C_1 \text{ else } 0)$.

1.1 Dry Run

Case 1: $S$ = "101" ($N=3$)

  • $C_1 = 2$, $C_0 = 1$.
  • $\min(2, 1) = 1$. Length from pairs = $2 \times 1 = 2$. ("10")
  • $C_1 \neq C_0$? Yes. Add 1. Length = $2 + 1 = 3$. ("101")
  • Output: 3

Case 2: $S$ = "0000" ($N=4$)

  • $C_0 = 4$, $C_1 = 0$.
  • $\min(4, 0) = 0$. Length from pairs = 0.
  • $C_0 \neq C_1$? Yes. Add 1. Length = $0 + 1 = 1$. ("0")
  • Output: 1

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ String S
    
    count0 = 0
    count1 = 0
    
    # Count frequencies
    FOR char in S:
        IF char is '0':
            count0 = count0 + 1
        ELSE:
            count1 = count1 + 1
            
    # Calculate Result
    result = 2 * MIN(count0, count1)
    
    IF count0 != count1:
        result = result + 1
        
    PRINT result
  • Time Complexity: $O(N)$ to iterate through the string.
  • Space Complexity: $O(1)$ auxiliary space.
Why is the answer $2 \cdot \min(C_0, C_1) + 1$ when counts are unequal?

A: We match pairs of '0' and '1' (giving length $2 \cdot \min$), and we can append exactly one of the remaining extra characters to the end. Adding more would create adjacent duplicates.
