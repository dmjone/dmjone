1. Problem Statement & Logic

Problem: Given potatoes harvested $x$ and $y$. Find minimum extra potatoes $z$ ($z > 0$) such that total $x+y+z$ is a prime number.

Logic:

  • Calculate sum $S = x + y$.
  • Iterate $z$ starting from 1 ($z=1, 2, 3 \dots$).
  • Check if $S+z$ is prime.
  • The first $z$ that satisfies the condition is the answer.
  • Primality check: Loop from 2 to $\sqrt{N}$. If divides, not prime.

1.1 Dry Run

Example: $x=1, y=3 \to S=4$.

  • $z=1$: $S+1=5$ (Prime). Output: 1.

Example: $x=4, y=4 \to S=8$.

  • $z=1$: 9 (Not prime).
  • $z=2$: 10 (Not prime).
  • $z=3$: 11 (Prime). Output: 3.

1.2 Pseudocode Solution


FUNCTION isPrime(n):
    IF n <= 1: RETURN False
    FOR i FROM 2 TO SQRT(n):
        IF n % i == 0: RETURN False
    RETURN True

READ T
FOR i FROM 1 TO T:
    READ x, y
    sum_xy = x + y
    z = 1
    WHILE True:
        IF isPrime(sum_xy + z):
            PRINT z
            BREAK
        z = z + 1
  • Time Complexity: $O(\sqrt{S} \times \text{gap})$, where gap is distance to next prime (small).
  • Space Complexity: $O(1)$.
Why do we start iterating $z$ from 1?

A: The problem specifies finding "extra" potatoes, implying $z > 0$. We need the smallest positive integer added to the sum to make it prime.
