1.1 Dry Run

Example: $S=$ "10100"

  • Count(0) = 3. Count(1) = 2.
  • Sorted: "00011". Output: "00011".

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ S
    
    count0 = 0
    count1 = 0
    FOR char in S:
        IF char == '0': count0 = count0 + 1
        ELSE: count1 = count1 + 1
        
    # Print count0 times '0' then count1 times '1'
    PRINT REPEAT('0', count0) + REPEAT('1', count1)
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(N)$.
Can we always completely sort the binary string using these reversals?

A: Yes! The operation (reversing a prefix ending at a transition) is powerful enough to generate any permutation of the string. Thus, we can always transform it into the sorted (lexicographically smallest) form.
