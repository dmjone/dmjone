1. Problem Statement & Logic

Problem: Chef sets a goal to eat $K$ grams of protein every day for $N$ days. On the $i$-th day, he buys $A_i$ grams. He can store leftover protein. Determine if he can meet the daily goal. If not, find the first day he fails.

Logic:

  • Maintain a variable `store` for protein. Initially 0.
  • Iterate through days $1$ to $N$:
    • Add bought protein to store: `store = store + A[i]`.
    • Check if `store >= K`.
    • If yes: Eat $K$ ($store = store - K$). Continue to next day.
    • If no: He fails on this day. Output "NO " + current_day_number. Terminate/Break.
  • If loop finishes without failure, output "YES".

1.1 Dry Run

Example: $N=3, K=5$. $A = [7, 3, 6]$.

  • Day 1: Buy 7. Store=7. Need 5. $7 \ge 5$. Eat 5. Left=2.
  • Day 2: Buy 3. Store=$2+3=5$. Need 5. $5 \ge 5$. Eat 5. Left=0.
  • Day 3: Buy 6. Store=$0+6=6$. Need 5. $6 \ge 5$. Eat 5. Left=1.
  • All days passed. Output: YES.

Example: $N=3, K=4$. $A = [3, 5, 10]$.

  • Day 1: Buy 3. Store=3. Need 4. $3 < 4$. Fail.
  • Output: NO 1.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N, K
    READ Array A
    
    store = 0
    possible = TRUE
    fail_day = 0
    
    FOR j FROM 0 TO N-1:
        store = store + A[j]
        IF store >= K:
            store = store - K
        ELSE:
            possible = FALSE
            fail_day = j + 1
            BREAK
            
    IF possible:
        PRINT "YES"
    ELSE:
        PRINT "NO " + fail_day
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
What happens to the excess protein bought on a given day?

A: It is stored and carried over to the next day to help meet future nutritional goals.
