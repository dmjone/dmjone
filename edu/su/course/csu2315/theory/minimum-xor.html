Minimum XOR

2025, Dec 15
View Problem Online

1. Problem Statement & Logic

Problem: Given array $A$. Removing at most 1 element, minimize XOR sum of all elements.

Logic:

  • Calculate total XOR of all elements: $X = A_1 \oplus A_2 \oplus \dots \oplus A_N$.
  • Option 1: Remove nothing. Result $X$.
  • Option 2: Remove $A_i$. New XOR is $X \oplus A_i$ (since $x \oplus x = 0$, removing is same as XORing again).
  • Iterate through all $A_i$ and check if $X \oplus A_i < \text{current\_min}$.
  • Update min.

1.1 Dry Run

Example: $A=[2, 4, 3, 6]$

  • Total XOR: $2 \oplus 4 \oplus 3 \oplus 6 = 6 \oplus 3 \oplus 6 = 5 \oplus 6 = 3$. (Wait: $2(010) \oplus 4(100) = 110(6)$. $6 \oplus 3(011) = 101(5)$. $5 \oplus 6(110) = 011(3)$). Base: 3.
  • Remove 2: $3 \oplus 2 = 1$. (Min so far 1).
  • Remove 4: $3 \oplus 4 = 7$.
  • Remove 3: $3 \oplus 3 = 0$. (Min so far 0).
  • Remove 6: $3 \oplus 6 = 5$.
  • Min is 0. Output: 0.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ A (array)
    
    total_xor = 0
    FOR x in A:
        total_xor = total_xor XOR x
        
    min_xor = total_xor
    
    FOR x in A:
        current_xor = total_xor XOR x
        IF current_xor < min_xor:
            min_xor = current_xor
            
    PRINT min_xor
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
Why does calculating $TotalXOR \oplus A_i$ simulate removing $A_i$?

A: XOR is its own inverse ($x \oplus x = 0$). Since $TotalXOR$ includes $A_i$ (i.e., $\dots \oplus A_i \oplus \dots$), XORing it again with $A_i$ cancels it out, leaving the XOR sum of the remaining elements.
