1. Problem Statement & Logic

Problem: Chef must move $N$ boxes of weights $W_i$. Can carry max weight $K$. Must pick boxes in order $1 \dots N$. Find min trips.

Logic:

  • This is a greedy problem.
  • Iterate through boxes. Add current box weight to valid load if current load + $W_i \le K$.
  • If not, start a new trip. (Increment trips, reset load to $W_i$).
  • Check feasibility: If valid box $W_i > K$, impossible. Output -1.
  • Start with 1 trip (if $N > 0$).

1.1 Dry Run

Example: $K=10, W=[3, 4, 3, 5]$

  • Box 1 (3): Load 3.
  • Box 2 (4): Load 3+4=7.
  • Box 3 (3): Load 7+3=10. Full.
  • Box 4 (5): 10+5 > 10. New Trip. Load 5.
  • End. Trips = 1 (complete) + 1 (last) = 2. Wait, logic check:
    • Trip 1: [3, 4, 3]. Sum 10.
    • Trip 2: [5]. Sum 5.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N, K
    READ W (array)
    
    trips = 1
    current_load = 0
    possible = TRUE
    
    FOR w in W:
        IF w > K:
            possible = FALSE
            BREAK
            
        IF current_load + w <= K:
            current_load = current_load + w
        ELSE:
            trips = trips + 1
            current_load = w
            
    IF possible:
        PRINT trips
    ELSE:
        PRINT -1
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
When must we immediately start a new trip?

A: When adding the next box's weight would exceed the weight limit $K$ for the current trip.
