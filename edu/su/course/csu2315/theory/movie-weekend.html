1. Problem Statement & Logic

Problem: Given $N$ movies with Length $L$ and Rating $R$. Choose index $i$ maximizing $L_i \times R_i$. Tie-break: Max $R_i$. Tie-break 2: Min index.

Logic:

  • Iterate through all movies $i=1 \dots N$.
  • Maintain `best_index`, `max_LR`, `max_R`.
  • Calculate `curr_LR = L[i] * R[i]`.
  • Update if:
    • `curr_LR > max_LR`
    • OR `curr_LR == max_LR` AND `R[i] > max_R`
    • OR `curr_LR == max_LR` AND `R[i] == max_R` (Index is automatically handled by iterating 1 to $N$ and only updating on strict better condition or skipping if same). Wait, problem says min index. By iterating $1 \dots N$ and valid update only on strict conditions, we naturally keep the first best.
    • Correct logic: Update if better. If equal LR and equal R, keep old (smaller index).

1.1 Dry Run

Example: $N=2$. $L=[2, 1], R=[3, 6]$.

  • Movie 1: $2 \times 3 = 6$. Best.
  • Movie 2: $1 \times 6 = 6$. Equal LR. Check R: $6 > 3$. Better. Best is 2.
  • Output: 2.

1.2 Pseudocode Solution

READ T
FOR i FROM 1 TO T:
    READ N
    READ L (array)
    READ R (array)
    
    max_lr = -1
    max_r = -1
    best_index = -1
    
    FOR j FROM 0 TO N-1:
        prod = L[j] * R[j]
        rating = R[j]
        
        IF prod > max_lr:
            max_lr = prod
            max_r = rating
            best_index = j + 1
        ELSE IF prod == max_lr:
            IF rating > max_r:
                max_r = rating
                best_index = j + 1
                
    PRINT best_index
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
