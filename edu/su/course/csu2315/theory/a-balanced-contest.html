1. Problem Statement & Logic

Problem: A contest is balanced if it has exactly 1 "cakewalk" problem (solved by $\ge P/2$ participants) and exactly 2 "hard" problems (solved by $\le P/10$ participants). Given number of solves for each problem, determine if the contest is balanced.

Logic:

  • Iterate through the solved counts of all problems.
  • Count how many problems satisfy `solved >= P / 2` (let's call it `count_cakewalk`).
  • Count how many problems satisfy `solved <= P / 10` (let's call it `count_hard`).
  • If `count_cakewalk == 1` AND `count_hard == 2`, output "yes".
  • Otherwise, output "no".

1.1 Dry Run

Example: $P=10$, Solves: [3, 5, 1, 1, 8]

  • Thresholds: Cakewalk $\ge 5$, Hard $\le 1$.
  • 3: No. 5: No (wait, $5 \ge 5$, yes cakewalk). 1: Hard. 1: Hard. 8: Cakewalk.
  • Cakewalk count: 2 (5, 8). Hard count: 2 (1, 1).
  • Not balanced (Cakewalk count $\neq$ 1). Output: no.

Example: $P=10$, Solves: [5, 1, 1]

  • Cakewalk count: 1 (5). Hard count: 2 (1, 1). Output: yes.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N, P
    READ So (array of solves)
    
    cakewalk = 0
    hard = 0
    
    FOR solves in So:
        IF solves >= P / 2:
            cakewalk = cakewalk + 1
        ELSE IF solves <= P / 10:
            hard = hard + 1
            
    IF cakewalk == 1 AND hard == 2:
        PRINT "yes"
    ELSE:
        PRINT "no"
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
What are the exact conditions for a "Balanced" contest?

A: Exactly 1 cakewalk problem (solved by $\ge P/2$) and exactly 2 hard problems (solved by $\le P/10$).
