Ciel and A-B Problem

2025, Dec 15
View Problem Online

1. Problem Statement & Logic

Problem: Calculate $A - B$. Then change exactly one digit of the result to make it wrong. (Input $A, B$).

Logic:

  • Compute diff = $A - B$.
  • To change one digit, we can simply look at the last digit (units place).
  • If the last digit is not 9, add 1.
  • If the last digit is 9, subtract 1.
  • This ensures:
    • The value changes (so it's wrong).
    • The number of digits remains the same (adding 1 to something ending in 8 or less won't carry over to a new digit count unless it was like 8. Wait, adding 1 to 8 becomes 9. Safe. Adding 1 to less than 9 is safe from carry potentially increasing length? Yes).
    • Subtracting 1 from 9 becomes 8. Safe.
    • Leading zeroes are avoided because result is positive $A-B$.

1.1 Dry Run

Example: $A=5858, B=1234$

  • Diff = 4624. Last digit 4.
  • Add 1. Result: 4625.

Example: $A=100, B=11$. Diff = 89. Last digit 9. Subtract 1 $\to$ 88.

1.2 Pseudocode Solution

READ A, B
diff = A - B
last_digit = diff % 10

IF last_digit == 9:
    diff = diff - 1
ELSE:
    diff = diff + 1
    
PRINT diff
  • Time Complexity: $O(1)$.
  • Space Complexity: $O(1)$.
