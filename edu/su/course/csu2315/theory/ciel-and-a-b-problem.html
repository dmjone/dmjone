1. Problem Statement & Logic

Problem: Given two integers $A$ and $B$, compute the value of $A - B$. Then print a wrong answer such that:

  • The answer is a positive integer.
  • The number of digits is the same as the correct answer.
  • Exactly one digit is different.
  • No leading zero is allowed.

Logic:

  • First, compute the correct difference: $diff = A - B$.
  • We must change only one digit of this number.
  • The safest digit to change is the last digit (units place) because it does not affect the number of digits.
  • If the last digit is not 9, increase it by 1.
  • If the last digit is 9, decrease it by 1.
  • This guarantees:
    • Exactly one digit is changed.
    • The value becomes incorrect.
    • The number of digits remains unchanged.
    • No leading zero is created, since only the last digit is modified.

1.1 Dry Run

Example 1: $A = 5858, B = 1234$

  • Correct difference: $5858 - 1234 = 4624$
  • Last digit = 4 (not 9)
  • Add 1 to last digit
  • Wrong answer = 4625

Example 2: $A = 100, B = 11$

  • Correct difference: $100 - 11 = 89$
  • Last digit = 9
  • Subtract 1 from last digit
  • Wrong answer = 88

1.2 Pseudocode Solution


READ A, B
diff = A - B
last_digit = diff % 10

IF last_digit == 9:
    diff = diff - 1
ELSE:
    diff = diff + 1

PRINT diff
  • Time Complexity: $O(1)$
  • Space Complexity: $O(1)$
Why do we subtract 1 if the last digit is 9?

A: To ensure we change exactly one digit. Adding 1 to 9 (e.g., 19 to 20) would change two digits due to carrying, which violates the requirement.
