1. Problem Statement & Logic

Problem: Alice and Bob play with string $S$. They alternate picking characters from $S$ and adding to their own strings $A$ and $B$. Can they make $A$ and $B$ equal? (Alice moves first).

Logic:

  • If Alice and Bob end up with identical strings, it means the final set of characters in $A$ is the same as in $B$.
  • This implies that for every character type (e.g., 'a', 'b'), the total count in $S$ must be even (split equally between $A$ and $B$).
  • Also, since they pick optimally/alternately, can they force the order? Actually, the problem simply asks if there EXIST a sequence of moves.
  • Since players cooperate (problem asks "is it possible", not "does Alice win"), they can just pick matching characters.
  • Condition: Counts of all characters in $S$ must be even. If $N$ is odd, impossible ($N$ must be even).
  • Answer: YES if all char counts are even. NO otherwise.
  • Wait, Alice moves first. They take turns. If counts are even, Alice takes $c$, Bob takes $c$. They can build identical strings reversed? No, order matters.
  • However, since they pick anytime from $S$, they can pick in any order. So if multisets are equal, they can build the same string.

1.1 Dry Run

Example: $S=$ "abab"

  • Freq: a:2, b:2. All even. Output: YES.

Example: $S=$ "abc"

  • Freq: a:1, b:1, c:1. Odd. Output: NO.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ S
    
    Freq = Map
    FOR char in S:
        Freq[char] = Freq[char] + 1
        
    possible = TRUE
    FOR count in Freq.values():
        IF count % 2 != 0:
            possible = FALSE
            BREAK
            
    IF possible:
        PRINT "YES"
    ELSE:
        PRINT "NO"
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
Why is the condition for equal strings simply that all character counts are even?

A: Alice and Bob cooperate to form identical strings. They can achieve this if and only if the characters in the source string $S$ can be split into two identical multisets (one for Alice, one for Bob). This requires every unique character to appear an even number of times.
