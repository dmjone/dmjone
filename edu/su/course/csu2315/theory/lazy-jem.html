1. Problem Statement & Logic

Problem: Jem has $N$ problems. Takes $M$ minutes for first problem. Strategy: Solve $(N+1)/2$ problems (if odd) or $N/2$ (if even). Then take break of $B$ mins. Then $M$ doubles ($2M$). Repeat until all solved. Find total time.

Logic:

  • While $N > 0$:
    • Problems to solve ($P$): If $N$ even, $P = N/2$. Else $P = (N+1)/2$.
    • Time for problems: $P \times M$.
    • Add to Total: $P \times M$.
    • Remaining problems: $N = N - P$.
    • If $N > 0$ (more problems left):
      • Add break time $B$.
      • Double rate: $M = 2 \times M$.
  • Note: Use 64-bit integers (long long) as time can exceed $2^{31}$.

1.1 Dry Run

Example: $N=9, B=1, M=2$

  • Round 1: $N=9$ (odd). Solve $(9+1)/2 = 5$. Time $5 \times 2 = 10$. Total=10. Rem $N=4$. Break ($+1 \to 11$). $M \to 4$.
  • Round 2: $N=4$ (even). Solve $4/2 = 2$. Time $2 \times 4 = 8$. Total $11+8=19$. Rem $N=2$. Break ($+1 \to 20$). $M \to 8$.
  • Round 3: $N=2$. Solve 1. Time $1 \times 8 = 8$. Total $20+8=28$. Rem $N=1$. Break ($+1 \to 29$). $M \to 16$.
  • Round 4: $N=1$. Solve 1. Time $1 \times 16 = 16$. Total $29+16=45$. Rem $N=0$. Stop.
  • Output: 45.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N, B, M
    total_time = 0
    
    WHILE N > 0:
        solved_count = 0
        IF N % 2 == 0:
            solved_count = N / 2
        ELSE:
            solved_count = (N + 1) / 2
            
        total_time = total_time + (solved_count * M)
        N = N - solved_count
        
        IF N > 0:
            total_time = total_time + B
            M = M * 2
            
    PRINT total_time
  • Time Complexity: $O(\log N)$.
  • Space Complexity: $O(1)$.
Why is the time calculation split into two parts (solving + break)?

A: The rate $M$ applies only to solving time. The break time $B$ is a fixed constant added after solving a batch. The rate doubles only *after* the break, so the order (Solve -> Add Time -> Break -> Change Rate) varies per step.
