1. Problem Statement & Logic

Problem: Given an array $A$. Choose two distinct elements $(x, y)$ such that sum of digits of their product $x \times y$ is maximum.

Logic:

  • Constraints are usually small ($N \le 100$). A brute force approach checking all pairs is feasible.
  • Logic:
    • Iterate through all pairs $(A[i], A[j])$ where $i < j$.
    • Calculate Product $P = A[i] \times A[j]$.
    • Calculate Sum of Digits of $P$, let it be $S$.
    • Track maximum $S$.
  • Why brute force? $O(N^2)$ for $N=100$ is $10,000$ iterations, which is very fast.

1.1 Dry Run

Example: $A = [9, 10, 11]$

  • Pairs:
    • (9, 10) $\to$ $90 \to$ Sum $9$.
    • (9, 11) $\to$ $99 \to$ Sum $18$.
    • (10, 11) $\to$ $110 \to$ Sum $2$.
  • Max Sum is 18. Output: 18.

1.2 Pseudocode Solution


FUNCTION digit_sum(num):
    sum = 0
    WHILE num > 0:
        sum = sum + (num % 10)
        num = num / 10
    RETURN sum

READ T
FOR i FROM 1 TO T:
    READ N
    READ Array A
    max_s = -1
    
    FOR j FROM 0 TO N-1:
        FOR k FROM j+1 TO N-1:
            product = A[j] * A[k]
            s = digit_sum(product)
            IF s > max_s:
                max_s = s
                
    PRINT max_s
  • Time Complexity: $O(N^2 \log (\text{Product}))$.
  • Space Complexity: $O(1)$.
Why is a brute force $O(N^2)$ solution acceptable here?

A: Because $N \le 100$, so $N^2 = 10,000$, which is well within the typical $10^8$ operations per second limit.
