1. Problem Statement & Logic

Problem: Given a permutation $P$. Allowed op: Swap adjacent elements. Goal: Move 1 to index 0 ($\text{P}[0]=1$) and $N$ to index $N-1$ ($\text{P}[N-1]=N$). Find min operations.

Logic:

  • Find current index of 1, say $idx_1$.
  • Moves to bring 1 to start = $idx_1$. (Each swap moves it left by 1).
  • Find current index of $N$, say $idx_N$.
  • Moves to bring $N$ to end = $(N-1) - idx_N$.
  • Total moves = $idx_1 + ((N-1) - idx_N)$.
  • Correction: The moves might interfere.
    • If $idx_1 < idx_N$: Moving 1 left doesn't affect $N$'s position relative to end (or $N$ moves right, 1 moves left, they don't cross). Just sum the moves.
    • If $idx_1 > idx_N$: To move 1 to start, it must cross $N$. When we swap 1 and $N$, both get closer to their targets. We save 1 move. Total = moves - 1.

1.1 Dry Run

Example: $P = [3, 2, 1]$, $N=3$

  • 1 is at index 2. $idx_1 = 2$. Moves for 1: 2.
  • 3 is at index 0. $idx_N = 0$. Moves for 3: $2-0=2$.
  • $idx_1 > idx_N$ ($2 > 0$). They cross.
  • Total = $2 + 2 - 1 = 3$.

Example: $P = [1, 3, 2]$, $N=3$

  • 1 at 0 ($idx_1=0$). Moves: 0.
  • 3 at 1 ($idx_N=1$). Moves: $2-1=1$.
  • $0 < 1$. No cross.
  • Total: $0+1=1$.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ P
    
    idx_1 = -1
    idx_N = -1
    
    FOR j FROM 0 TO N-1:
        IF P[j] == 1:
            idx_1 = j
        IF P[j] == N:
            idx_N = j
            
    moves = idx_1 + (N - 1 - idx_N)
    
    IF idx_1 > idx_N:
        moves = moves - 1
        
    PRINT moves
  • Time Complexity: $O(N)$ to find indices.
  • Space Complexity: $O(1)$.
How do we calculate the total moves efficiently?

A: Calculate moves for 1 ($idx_1$) and moves for N ($N-1-idx_N$). Add them up. Crucially, if $idx_1 > idx_N$, subtract 1 because swapping them helps both targets simultaneously.
