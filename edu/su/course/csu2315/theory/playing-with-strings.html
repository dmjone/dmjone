1. Problem Statement & Logic

Problem: Given binary strings $S$ and $R$. You can swap any two characters in $S$ any number of times. Can you make $S$ equal to $R$?

Logic:

  • If we can swap any two characters, we can arbitrarily reorder $S$.
  • This means we can transform $S$ to $R$ if and only if they have the same multiset of characters.
  • Since strings are binary, this simply means they must have the same number of '1's (and consequently '0's).
  • Count '1's in $S$ and $R$. If equal, output YES. Else NO.

1.1 Dry Run

Example: $S=110, R=101$

  • Sum(S) = 2. Sum(R) = 2. Equal. Output: YES.

Example: $S=110, R=000$

  • Sum(S) = 2. Sum(R) = 0. Not Equal. Output: NO.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ S
    READ R
    
    countS = 0
    FOR char in S:
        IF char == '1': countS = countS + 1
        
    countR = 0
    FOR char in R:
        IF char == '1': countR = countR + 1
        
    IF countS == countR:
        PRINT "YES"
    ELSE:
        PRINT "NO"
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
Why does the ability to swap *any* two characters imply we can match any string with the same character counts?

A: Swapping any two characters allows us to place any specific character into any desired position. This means the specific order is irrelevant; only the frequency of each character (the multiset) matters.
