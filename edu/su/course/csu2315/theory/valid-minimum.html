Valid Minimum

1. Problem Statement & Logic

Problem: Given 3 integers representing $\min(A, B)$, $\min(B, C)$, and $\min(C, A)$. Do there exist integers $A, B, C$ satisfying these minimums?

Logic:

  • Let given values be $X, Y, Z$.
  • Consider the set of minimums. The smallest minimum must appear at least twice. Why?
  • Suppose $A$ is smallest of all three. Then $\min(A, B)=A$ and $\min(C, A)=A$. So $A$ appears in two pairs.
  • Thus, in the sorted list of given values, the two smallest values must be equal.
  • Also, does the third value matter? It must be $\ge$ the smallest value (obvious).
  • Conditions:
    • Sort $X, Y, Z$ as $S_1, S_2, S_3$ ($S_1 \le S_2 \le S_3$).
    • We must have $S_1 == S_2$.
  • Wait, is that sufficient? Examples:
    • (2, 2, 5): $A=2, B=5, C=?. \min(A, C)=2$. $\min(B, C)=2 \to \min(5, C)=2 \to C=2$. $\min(A, B)=2 \to \min(2, 5)=2$ (ok).
    • So (2, 2, 5) works. Tuple (2, 5, 2).
    • (2, 3, 5): Smallest is 2. Must divide others? No.
  • Conclusion: YES if the minimum value among the three inputs appears at least twice.

1.1 Dry Run

Example: $X=3, Y=4, Z=3$

  • Sorted: 3, 3, 4. Min is 3. Count is 2. Output: YES.

Example: $X=5, Y=3, Z=9$

  • Sorted: 3, 5, 9. Min is 3. Count is 1. Output: NO.

1.2 Pseudocode Solution

READ T
FOR i FROM 1 TO T:
    READ A_min, B_min, C_min
    
    minimum = MIN(A_min, B_min, C_min)
    count = 0
    
    IF A_min == minimum: count = count + 1
    IF B_min == minimum: count = count + 1
    IF C_min == minimum: count = count + 1
    
    IF count >= 2:
        PRINT "YES"
    ELSE:
        PRINT "NO"
  • Time Complexity: $O(1)$.
  • Space Complexity: $O(1)$.
