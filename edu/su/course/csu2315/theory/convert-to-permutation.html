Convert to permutation

1. Problem Statement & Logic

Problem: Given array $A$ of size $N$. Can we make it a permutation $1 \dots N$ by increasing elements? If yes, min operations?

Logic:

  • We need to map the given numbers to the target set $\{1, 2, \dots, N\}$.
  • Sort observed array $A$ ascending.
  • Compare $A[i]$ with target value $i+1$ (for $i$ from $0$ to $N-1$).
  • Since we can only *increase* values:
    • If $A[i] > i+1$: Impossible (we can't decrease). Output -1.
    • Otherwise, operations needed = $(i+1) - A[i]$.
  • Sum operations.

1.1 Dry Run

Example: $N=3, A=[1, 1, 1]$

  • Sort A: $[1, 1, 1]$. Targets: $[1, 2, 3]$.
  • $1 \le 1$: Ops 0.
  • $1 \le 2$: Ops 1.
  • $1 \le 3$: Ops 2.
  • Total: 3. Output: 3.

Example: $N=3, A=[4, 1, 2]$

  • Sort A: $[1, 2, 4]$. Targets: $[1, 2, 3]$.
  • $1 \le 1$: Ops 0.
  • $2 \le 2$: Ops 0.
  • $4 > 3$: Impossible. Output: -1.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ A (array)
    SORT(A)
    
    total_ops = 0
    possible = TRUE
    
    FOR j FROM 0 TO N-1:
        target = j + 1
        IF A[j] > target:
            possible = FALSE
            BREAK
        ELSE:
            total_ops = total_ops + (target - A[j])
            
    IF possible:
        PRINT total_ops
    ELSE:
        PRINT -1
  • Time Complexity: $O(N \log N)$ (sorting).
  • Space Complexity: $O(1)$.
Why must we check if checking each number against its target possible?

A: Because we can only increase values. If $A[i]$ is already greater than its target $i+1$, we cannot reduce it, making the permutation impossible.
