1. Problem Statement & Logic

Problem: Three parties A, B, C contested. Given votes $X_A, X_B, X_C$. A party wins if it receives strictly more than 50 votes. Output winning party or "NOTA" if none wins. Total votes = 101.

Logic:

  • Input: $X_A, X_B, X_C$.
  • Check if $X_A > 50 \to$ Print 'A'.
  • Check if $X_B > 50 \to$ Print 'B'.
  • Check if $X_C > 50 \to$ Print 'C'.
  • Else $\to$ Print "NOTA".
  • Since total is 101, only one party can possibly get > 50. So exclusive checks are fine.

1.1 Dry Run

Example: 80 10 11

  • $X_A = 80$. $80 > 50$? Yes. Output: A.

Example: 50 20 31

  • $X_A = 50$. $50 > 50$? No.
  • $X_B = 20$. $20 > 50$? No.
  • $X_C = 31$. $31 > 50$? No.
  • Output: NOTA.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ XA, XB, XC
    
    IF XA > 50:
        PRINT "A"
    ELSE IF XB > 50:
        PRINT "B"
    ELSE IF XC > 50:
        PRINT "C"
    ELSE:
        PRINT "NOTA"
  • Time Complexity: $O(1)$.
  • Space Complexity: $O(1)$.
Why don't we need to check if multiple parties have > 50 votes?

A: The total number of votes is 101. It is mathematically impossible for two parties to both have $>50$ votes (since $51+51=102 > 101$). Thus, exclusive checks are safe.
