ATM Machine

1. Problem Statement & Logic

Problem: There are $N$ people in a queue for an ATM that initially has $K$ units of money. Each person $i$ wants to withdraw $A_i$ amount. If the ATM has enough money ($K \ge A_i$), the transaction is successful, and the ATM balance decreases by $A_i$. If not, the transaction fails, and the balance remains the same. Determine the outcome (1 for success, 0 for failure) for each person in order.

Logic: This is a direct simulation problem. We process the requests one by one.

  • Maintain a variable current balance, initially $K$.
  • For each request $A_i$:
    • If $current \ge A_i$: Output 1, $current = current - A_i$.
    • Else: Output 0. (Do not subtract).

1.1 Dry Run

Example: $N=5, K=10$. Requests $A = [3, 5, 3, 2, 1]$.

  • Person 1 (Want 3): $10 \ge 3$. Success (1). $K = 10 - 3 = 7$.
  • Person 2 (Want 5): $7 \ge 5$. Success (1). $K = 7 - 5 = 2$.
  • Person 3 (Want 3): $2 < 3$. Fail (0). $K = 2$.
  • Person 4 (Want 2): $2 \ge 2$. Success (1). $K = 2 - 2 = 0$.
  • Person 5 (Want 1): $0 < 1$. Fail (0). $K = 0$.
  • Final Output: 11010

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N, K
    READ Array A
    
    FOR j FROM 0 TO N-1:
        request = A[j]
        
        IF K >= request:
            PRINT "1" (without newline)
            K = K - request
        ELSE:
            PRINT "0" (without newline)
            
    PRINT newline
  • Time Complexity: $O(N)$ per test case.
  • Space Complexity: $O(1)$ auxiliary space.
Does a failed transaction affect the ATM balance?

A: No. If a person asks for more than what's available ($K < A_i$), the transaction fails (Output 0) and the balance $K$ remains unchanged for the next person.
