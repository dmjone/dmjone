1. Problem Statement & Logic

Problem: Playlist $A$ has $N$ songs. "Uncle Johny" is at index $K$ (1-indexed). Sort the playlist by length. Find the new position of "Uncle Johny".

Logic:

  • Identify the length of "Uncle Johny" song: `uj_len = A[K-1]`.
  • Sort the entire array $A$ in ascending order.
  • Find the index of `uj_len` in the sorted array. Add 1 for 1-based indexing.
  • Alternatively, just count how many songs strictly smaller than `uj_len` exist. Position is `count + 1`. (Assuming unique lengths? Problem says unique integer length).

1.1 Dry Run

Example: $A=[1, 3, 4, 2], K=2$

  • Song at index 2 (1-based) is 3. `uj_len = 3`.
  • Sorted Array: $[1, 2, 3, 4]$.
  • Index of 3 is 2 (0-based) $\to$ position 3. Output: 3.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ A (array)
    READ K
    
    uj_len = A[K-1]
    SORT(A)
    
    FOR j FROM 0 TO N-1:
        IF A[j] == uj_len:
            PRINT j + 1
            BREAK
  • Time Complexity: $O(N \log N)$ (sorting).
  • Space Complexity: $O(1)$ (in-place sort).
Why do we search for the length value after sorting?

A: Sorting rearranges the songs by length. Since we need the new rank of the specific song "Uncle Johny" (identified by its unique length), we must locate where its length value ended up in the sorted sequence.
