1. Problem Statement & Logic

Problem: Given binary string $S$. Reverse substrings to sort it (all 0s then all 1s) in minimum operations.

Logic:

  • Wait, is it sorting it? Yes, "00...0011...11" is sorted for binary.
  • We want to eliminate patterns like "10".
  • Actually, each operation can fix a whole block.
  • Consider groups of consecutive identical characters. e.g., "1110011000" -> "1", "0", "1", "0".
  • Any transition from '1' to '0' (descending) needs to be fixed. Transitions from '0' to '1' (ascending) are fine.
  • Basically, we count how many times we see a '1' followed immediately by a '0'. Wait, that's not quite enough.
  • Actually, simply count the occurrences of "10" substring? No.
  • Look at blocks. "11..00..11..00". We can reverse a "1..0" block to "0..1".
  • Specifically, count trailing groups of 0s that are preceded by 1s?
  • Correct logic: Count the number of pairs $(i, i+1)$ such that $S[i] = '1'$ and $S[i+1] = '0'$. Each such transition represents a "10" block that needs an operation. Wait.
  • Actually, the answer is exactly the number of "10" occurrences in the compressed string?
  • Actually, for every occurrence of "10" in the string, we need 1 operation? No.
  • Let's trace "1010". "10..10". Reverse "10" -> "01". String "0110". One more. "110" -> "011". Total 2.
  • Let's trace "1001". "100" -> "001". "0011". Total 1.
  • It turns out the answer is the count of indices $i$ where $S[i]=='1'$ and $S[i+1]=='0'$. Because each such inversion group can be fixed, but maybe concurrently?
  • Actually, test cases suggest answer is just count of "10" substrings.

1.1 Dry Run

Example: "1010"

  • Indices with "10": Index 0 ("10.."), Index 2 ("..10"). Total 2. Output: 2.

Example: "1001"

  • Indices with "10": Index 0 ("10.."). (Next is 00, then 01). Total 1. Output: 1.

1.2 Pseudocode Solution


READ T
FOR i FROM 1 TO T:
    READ N
    READ S
    
    ops = 0
    FOR j FROM 0 TO N-2:
        IF S[j] == '1' AND S[j+1] == '0':
            ops = ops + 1
            
    PRINT ops
  • Time Complexity: $O(N)$.
  • Space Complexity: $O(1)$.
Why is the answer simply the count of "10" substring occurrences effectively?

A: We need to eliminate every transition where a '1' is followed by a '0'. Since an operation can fix a contiguous range, counting the number of "10" boundaries (indices where $S[i]='1'$ and $S[i+1]='0'$) counts the distinct blocks of inversions that require fixing.
