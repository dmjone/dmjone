Network Security: SSL/TLS

SSL (Secure Sockets Layer) and TLS (Transport Layer Security) are cryptographic protocols that provide secure communication over a network. TLS is the successor to SSL and is more secure. They use encryption to ensure:

  • Confidentiality: Data is encrypted to prevent eavesdropping.
  • Integrity: Hashing mechanisms ensure data is not tampered with during transmission.
  • Authentication: Digital certificates validate the server and, optionally, the client.

The handshake process in SSL/TLS establishes a secure session using asymmetric encryption (RSA, ECC) and then switches to faster symmetric encryption (AES) for data transfer.

Network Security: ARP

ARP (Address Resolution Protocol) resolves IPv4 addresses to MAC addresses within a local network. It is essential for devices to communicate at the data link layer. ARP operates as follows:

  • Request: A device broadcasts an ARP request to find the MAC address of a given IP.
  • Reply: The device with the matching IP responds with its MAC address.

However, ARP is vulnerable to spoofing attacks, where an attacker sends forged ARP messages to associate their MAC address with a target's IP, potentially leading to man-in-the-middle attacks.

Sliding Window Protocol

The Sliding Window Protocol ensures reliable data transmission in TCP. It uses a window to control the flow of data and manage congestion. Key concepts:

  • Sender Window: Limits the number of unacknowledged packets the sender can transmit.
  • Receiver Window: Indicates how much data the receiver can handle.
  • Acknowledgments: The receiver sends ACKs for received packets, allowing the sender to slide the window forward.

This protocol improves efficiency by allowing continuous transmission of packets without waiting for individual acknowledgments, reducing idle time.

Why IP Addresses Are Used Instead of MAC Addresses for Data Transmission

While MAC addresses are unique to every hardware and used at the data link layer, IP addresses are essential for the following reasons:

  • Scalability: MAC addresses work only within a local network (LAN). IP addresses provide a hierarchical addressing scheme, enabling routing across vast, interconnected networks (WAN, Internet).
  • Global Reach: Routers operate at the network layer and cannot interpret MAC addresses. IP addresses are used to route data between different networks.
  • Logical Addressing: IP addresses allow dynamic assignment and logical grouping of devices, whereas MAC addresses are fixed and tied to hardware.
  • Network Segmentation: IP addresses enable subnetting and efficient traffic management, which is not possible with flat MAC address schemes.

Therefore, IP addresses are necessary for routing and managing data across multiple networks, while MAC addresses function only within a single local network.
