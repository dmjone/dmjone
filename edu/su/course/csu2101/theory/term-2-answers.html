In context of the above tables, write SQL queries to get the following output:

  1. Display the Employee names along with their manager’s name.
  2. Display the details of the employee who is getting the maximum salary in the organization.
  3. Display the details of the department for which the minimum-salary employee is working.
  4. Display the details of employees who are working in the same department for which ‘MILLER’ is working.
  5. Display the details of employees who are getting more salary than all those employees who are designated as MANAGER.

Section A Answers

1. Employee names with their manager’s name
SELECT 
  e.ENAME AS Employee,
  m.ENAME AS Manager
FROM EMP e
LEFT JOIN EMP m
  ON e.MGR = m.EMPNO;
2. Details of the employee with the maximum salary
SELECT *
FROM EMP
WHERE SAL = (SELECT MAX(SAL) FROM EMP);
3. Details of the department where the minimum-salary employee works
SELECT DISTINCT d.DEPTNO, d.DNAME, d.LOC
FROM DEPT d
JOIN EMP e ON d.DEPTNO = e.DEPTNO
WHERE e.SAL = (SELECT MIN(SAL) FROM EMP);
4. Employees working in the same department as ‘MILLER’
SELECT *
FROM EMP
WHERE DEPTNO = (
  SELECT DEPTNO
  FROM EMP
  WHERE ENAME = 'MILLER'
);
5. Employees earning more than all Managers
SELECT *
FROM EMP
WHERE SAL > ALL (
  SELECT SAL
  FROM EMP
  WHERE JOB = 'MANAGER'
);

Section B Answers

1. Difference between GET and POST methods

GET

  • Appends form data to URL as query string (e.g. ?user=alice&age=23).
  • Limited in length (~2 000 characters).
  • Data visible in browser history and logs (not secure for sensitive info).
  • Idempotent (safe to repeat).

POST

  • Sends data in HTTP request body.
  • No practical size limit.
  • Not shown in URL (more secure for credentials).
  • Not idempotent by default (used for inserts/updates).
2. Determining the form submission method in PHP
<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // Handle form data sent via POST
    $name = $_POST['name'];
    echo "Hello, $name (via POST)";
} else {
    // Handle GET (initial form load or GET-submission)
    $name = $_GET['name'] ?? '';
}
?>

3. Subquery vs. Inline View (Derived Table) in MySQL
  • Subquery: A query nested in SELECT, WHERE or HAVING. Returns a single value or a set to filter the parent query. 
    SELECT ENAME, SAL
FROM EMP
WHERE SAL > (
  SELECT AVG(SAL)
  FROM EMP
);
  • Inline View (Derived Table): A subquery in the FROM clause, treated as a temporary table with an alias. 
    SELECT d.DEPTNO, d.AVG_SAL
FROM (
  SELECT DEPTNO, AVG(SAL) AS AVG_SAL
  FROM EMP
  GROUP BY DEPTNO
) AS d
WHERE d.AVG_SAL > 3000;
4. What is a View in a Database?

A view is a virtual table defined by a stored SELECT query.

  • No data stored separately; it presents data from underlying tables.
  • Simplifies complex queries and hides sensitive columns.
  • Can be used like a table in SELECT, JOIN, UPDATE, etc. (updatable if simple).
CREATE VIEW SalesSummary AS
SELECT DEPTNO, COUNT(*) AS EmpCount, AVG(SAL) AS AvgSal
FROM EMP
WHERE JOB = 'SALESMAN'
GROUP BY DEPTNO;

Section C Answers

1. PHP Sessions

Definition: A session stores user-specific data on the server across multiple page requests.

How it works:

  • Server generates a session ID and sends it as a cookie (PHPSESSID) to the client.
  • On each request, PHP reads the session ID and loads the associated data.

Basic usage:

<?php
session_start();

// Store data in session
$_SESSION['user'] = 'alice';
$_SESSION['last_login'] = time();

// Retrieve session data
echo "Welcome, " . $_SESSION['user'];

// Destroy session (logout)
session_unset();
session_destroy();
?>

When to use:

  • Login systems, shopping carts, user preferences.
  • Keeps data secure on server (not exposed in URLs).
2. SQL Joins and Their Types

A join combines rows from two or more tables based on a related column.

  • INNER JOIN: Returns rows with matching keys in both tables. 
    SELECT e.ENAME, d.DNAME
FROM EMP e
INNER JOIN DEPT d
  ON e.DEPTNO = d.DEPTNO;
  • LEFT (OUTER) JOIN: All rows from the left table, plus matched rows from the right; unmatched right columns are NULL. 
    SELECT e.ENAME, d.DNAME
FROM EMP e
LEFT JOIN DEPT d
  ON e.DEPTNO = d.DEPTNO;
  • RIGHT (OUTER) JOIN: All rows from the right table, plus matched rows from the left; unmatched left columns are NULL. 
    SELECT e.ENAME, d.DNAME
FROM EMP e
RIGHT JOIN DEPT d
  ON e.DEPTNO = d.DEPTNO;
  • CROSS JOIN: Cartesian product: every row of one table with every row of the other. 
    SELECT e.ENAME, d.LOC
FROM EMP e
CROSS JOIN DEPT d;
  • SELF JOIN: Joining a table to itself (useful for hierarchical data). 
    SELECT e1.ENAME AS Employee, e2.ENAME AS Manager
FROM EMP e1
LEFT JOIN EMP e2
  ON e1.MGR = e2.EMPNO;
