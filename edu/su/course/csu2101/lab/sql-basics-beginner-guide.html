Practical 1: SQL for Beginners - Data Manipulation and Retrieval

2025, March 17
This document is intended to help you understand the practical actions performed by each part of the SQL code performed in this practical.

This explanation is designed for beginners with no prior knowledge of SQL, so that when you read the actual code later, you understand exactly what each part is meant to do.

1. Data Modification and Retrieval

UPDATE Statement

  • What It Does:
    The UPDATE command is used to change existing data in a table. In our example, it sets the department number (deptno) to a null value for the employee whose name matches a specific condition.
  • How It Works:
    Imagine you have a list of employees. If you want to “erase” the department information for the employee named 'KING', you run an UPDATE command that tells the database, “For every employee with the name 'KING', remove (or set to null) the department number.”

SELECT Statement

  • What It Does: The SELECT statement is used to retrieve data from one or more tables.
  • How It Works: Think of SELECT as asking the database, “Show me these pieces of information.” You can ask for all the data (all columns and rows) or just specific columns, and you can also choose to show only rows that meet certain conditions.

2. Filtering and Pattern Matching

WHERE Clause

  • What It Does:
    The WHERE clause filters the data by specifying conditions that the rows must meet.
  • How It Works:
    It’s like using a filter on a spreadsheet. For example, if you only want to see employees whose salary is greater than a specific amount, you would include a WHERE clause to check that condition.

LIKE Operator with Wildcards

  • What It Does:
    The LIKE operator allows you to search for patterns within text.
  • How It Works:
    • The % symbol represents any sequence of characters (it’s like saying “anything goes here”).
    • The _ symbol represents any single character.
      For example, ename like 'M_R_IN' means you are looking for names that start with "M," have any single character in the second spot, then "R," another single character, and end with "IN" (which would match "MARTIN").

3. Sorting Results

ORDER BY Clause

  • What It Does:
    The ORDER BY clause arranges the results in a specific order—either ascending (smallest to largest, or alphabetically) or descending (largest to smallest, or reverse alphabetical).
  • How It Works:
    Imagine a list of employees. If you want to see them ordered by their salary or by their names, you would use ORDER BY to specify that order. You can even order by a column's number (for example, order by the second column in the result).

4. Grouping and Aggregation

GROUP BY Clause

  • What It Does:
    GROUP BY collects rows that have the same values in specified columns into summary rows—like grouping employees by department.
  • How It Works:
    Once the rows are grouped, you can use aggregate functions to calculate totals, averages, counts, etc., for each group.

Aggregate Functions

  • SUM, AVG, MAX, MIN, COUNT:
    These functions perform calculations on sets of rows.
    • SUM: Adds up all the values in a column.
    • AVG: Calculates the average value.
    • MAX: Finds the highest value.
    • MIN: Finds the lowest value.
    • COUNT: Counts the number of rows.
  • How It Works:
    For example, if you group employees by department, you might want to know the total salary expense (SUM), the average salary (AVG), or how many employees there are (COUNT) in each department.

HAVING Clause

  • What It Does:
    The HAVING clause filters groups based on aggregate function results, similar to how WHERE filters individual rows.
  • How It Works:
    For example, after grouping employees by department, you can use HAVING to show only those departments where the average salary is above 2000.

5. Joining Tables

Inner Join

  • What It Does:
    An inner join combines rows from two tables only when there is a match in both tables based on a related column.
  • How It Works:
    Imagine two lists—one for employees and one for departments. An inner join will show only the employees who belong to a department that exists in the department list.

Outer Joins (Left, Right, Full)

  • Left Outer Join:
    Returns all rows from the left table and the matched rows from the right table. If there’s no match, the right side will show null.
  • Right Outer Join:
    Returns all rows from the right table and the matched rows from the left table.
  • Full Outer Join:
    Returns rows when there is a match in either table, showing nulls where no match exists.
  • How It Works:
    These joins help you see all data from one or both tables, even if some records don’t have a corresponding match in the other table.

Natural Join

  • What It Does:
    A natural join automatically joins tables based on columns with the same name and compatible data types in both tables.
  • How It Works:
    It’s like a shortcut where you don’t have to specify the join condition manually; the database figures it out based on common column names.

Self Join

  • What It Does:
    A self join is when you join a table with itself. This is useful for finding relationships within the same table, such as linking employees with their managers.
  • How It Works:
    For example, if each employee record contains a manager’s employee number, you can join the table to itself to create a list that shows each employee alongside their manager’s name.

6. Subqueries

Scalar Subqueries

  • What It Does:
    A subquery is a query nested inside another query. Scalar subqueries return a single value that is then used by the outer query.
  • How It Works:
    Think of it as a mini-question that provides an answer used in a bigger question. For example, you might want to compare each employee's salary to the highest salary in the table. The subquery finds the maximum salary, and then the outer query retrieves all rows that match this value.

Putting It All Together

When you see the code, you're looking at a set of SQL operations that together allow you to change, retrieve, summarize, and combine data from one or more tables. Each part of the code is a building block that, when combined, makes powerful data manipulation and retrieval possible:

  • Data is being modified (with UPDATE),
  • Filtered (using WHERE and LIKE),
  • Sorted (with ORDER BY),
  • Grouped (using GROUP BY along with aggregate functions and HAVING),
  • Joined (using various types of joins like inner, outer, and self joins), and
  • Queried with nested questions (via subqueries),
