1. Color: Two Branches

  • Full-color: real, captured by sensor
  • Pseudocolor: artificial, assigned to grayscale

Why three channels? Retina has three cone types (R, G, B).

2. Additive vs. Subtractive Mixing

  • Additive (light): R+G+B = White. Monitors, projectors.
  • Subtractive (ink): Y+M+C = Black. Printing.

Pair rule: additive secondaries (Y, C, M) are subtractive primaries.

3. RGB Model

3D Cartesian cube; axes $R, G, B \in [0,1]$.

  • $(0,0,0)$ = Black, $(1,1,1)$ = White
  • Main diagonal ($R=G=B$) = grayscale
  • Stored as three independent planes

Use: hardware-friendly. Cameras, monitors.

4. HSI Model

  • Hue (H): dominant wavelength (angle 0°–360°)
  • Saturation (S): purity (0 = gray, 1 = pure)
  • Intensity (I): brightness

Key win: decouples color from brightness.

Use: equalize $I$ only → no color shift.

5. Pseudocolor

Eye sees ~24 grays but thousands of colors. Map gray → color to expose detail.

5.1 Intensity Slicing

Divide $[0, L-1]$ into intervals; assign one color per interval.

Use: thermal, medical, satellite imagery.

6. Compression: The Goal

Eliminate redundancy in image data.

6.1 Three Types of Redundancy

  • Coding: codes longer than necessary → fix with variable-length
  • Spatial / Temporal: neighbors are correlated → fix with predictive/transform coding
  • Psychovisual: eye can't see it → safe to discard (lossy)

7. Fidelity Criteria

Objective (RMSE):

$$ e_{\text{rms}} = \sqrt{\tfrac{1}{MN} \sum [f - \hat{f}]^2} $$

Lower = better. Zero = lossless.

Subjective: human rating (1–5). RMSE doesn't capture perceptual importance.

8. Encoder Pipeline

  • Mapper: reduces spatial redundancy (DCT, wavelet, differences). Lossless.
  • Quantizer: reduces psychovisual redundancy. The only lossy stage.
  • Symbol coder: reduces coding redundancy (Huffman). Lossless.

Decoder reverses mapper and symbol coder. Quantizer has no inverse.

Rule: drop the quantizer → lossless compression.

9. Huffman Coding

Variable-length codes: short for frequent, long for rare.

9.1 Construction

  • Sort symbols by probability (descending)
  • Merge two lowest into one node (sum probabilities)
  • Repeat until one root remains
  • Assign 0/1 at each branch; trace root → symbol = code

9.2 Optimality

Smallest possible average code length for any zero-memory source.

Example: 6 symbols, 2.2 bits/symbol vs. 3 bits fixed → 26.7% savings.

10. Decision Engine (Exam Gold)

  • Hardware processing? → RGB
  • Enhance brightness without color shift? → HSI (process $I$ only)
  • Reveal grayscale detail? → Pseudocolor / intensity slicing
  • Lossless compression? → Skip quantizer; use Huffman
  • Lossy compression? → Add quantizer (the only lossy stage)
  • Measure quality? → RMSE (objective) + human rating (subjective)

Master rule: compression = identify redundancy, encode the rest.
