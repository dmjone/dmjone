Digital Image Processing

2026, March 8

Slide Page 1: What is an Image / Digital Image

An image is a 2-D function f(x,y) whose value at (x,y) is intensity. A digital image has discrete x,y and discrete intensity it is a matrix of pixels (finite M × N) each with a numeric value.

  • Key term: pixel = one matrix element, has location and intensity.
  • Exam short answer (2m): “Define digital image.” → A discrete 2-D function stored as an M×N array of pixel intensities.

Slide 4: Levels of Image Processing

Three levels:

  • Low-level: Input image → output image (noise removal, contrast, sharpening).
  • Mid-level: Image → attributes (edges, regions, segmentation, feature extraction).
  • High-level: Attributes → understanding (scene interpretation, recognition).

Why it matters: Test questions often ask to classify tasks into these levels and give examples.

Slide 24–25: Fundamental Pipeline of Image Processing

Typical ordered pipeline and purpose:

  • 1. Acquisition: capture & digitize (sensor + A/D). Good input reduces later work.
  • 2. Enhancement vs Restoration: Enhancement = subjective (make it look better). Restoration = objective (invert a known degradation model).
  • 3. Morphological processing: Shape-based operations (dilate, erode, skeletons) for structural info.
  • 4. Segmentation: Partition image into meaningful regions hardest but crucial for recognition.
  • 5. Recognition: Assign labels using extracted features.

Exam focus: Be able to list steps, say purpose of each, and contrast enhancement vs restoration (short 2–5m answers).

Slide 26: Components of an Image Processing System

Main blocks: image sensors, imaging system/hardware, storage, processing computer/software, display, network/cloud. Each block affects fidelity and throughput.

Tip: For 2–2.5m answers name components and one-line role (sensor: capture; storage: archive; display: visualization).

Slide 27–28: Elements of Visual Perception (Human Eye)

Important physiology linked to image design:

  • Cones: ~6–7M, central (fovea), color, high spatial detail (photopic).
  • Rods: ~75–150M, peripheral, sensitive in low light, no color (scotopic).
  • Human eye focuses by changing lens shape (focal length varies), unlike fixed-lens cameras.

Why tested: Relate sensors and sampling to human perception (why more gray levels matter for faces).

Slide 29–30: Image Sensing and Acquisition

Images come from illuminating the scene; sensor records reflected/transmitted energy. Key idea: illumination + scene interaction → measured signal.

  • Sensors convert energy to electrical signal → sampled and quantized.
  • Different modalities (visible, IR, X-ray, ultrasound) are same pipeline but different energy sources and sensors.

Slide 31: Image Formation Model

Two multiplicative components:

Illumination i(x,y) how much light hits the scene, and Reflectance r(x,y) how much is reflected. The observed intensity:

$f(x,y)=i(x,y)\,r(x,y)$

Reflectance typically in [0,1]; illumination ≥ 0. For transmissive imaging (X-ray) use transmission instead of reflectance.

Slide 32–33: Sampling and Quantization

Two steps to make a digital image from continuous scene:

  • Sampling (spatial): pick discrete coordinates x = 0..M−1, y = 0..N−1 → creates M×N grid of samples (pixels).
  • Quantization (intensity): map continuous intensity to finite L levels. For k bits per pixel: $$L=2^k$$. Example: 8-bit → 256 levels.
  • Image is stored as matrix A with entries a_{i,j} = f(i,j).

Exam tip (2–2.5m): Write formula L=2^k and define M,N,k briefly.

Slide 34–36: Spatial and Intensity Resolution

Definitions:

  • Spatial resolution: smallest discernible spatial detail governed by sampling rate (number of pixels per unit length). Low spatial resolution → jagged/blurred shapes.
  • Intensity (gray-level) resolution: smallest visible change in intensity governed by bits k (quantization). Low intensity resolution → false contouring posterization.

Tradeoffs & sensitivity: Objects with fine shape detail (crowd) need higher spatial sampling; smooth objects (face) need higher intensity resolution. The iso-preference curves describe equivalent perceived quality with different N and k.

Slide 36–38: Practical effects and digitization choices

Practical guidelines:

  • If shape detail is critical → increase sampling (N) more than bit depth.
  • If smooth shading is critical → increase intensity resolution (k) first.
  • Always check artifacts: jaggies (spatial under-sampling) vs false contours (intensity under-quantization).

Quick probable exam questions & one-line answers (mapped to marks)

Five × 2-mark short questions (likely)

  • Define digital image. Discrete 2-D function sampled to M×N pixels, each with intensity value.
  • What is a pixel? Smallest image element with position and intensity value.
  • Define sampling. Converting continuous coordinates to discrete grid points (M×N).
  • Define quantization. Mapping continuous intensities to L discrete levels; L = 2^k.
  • Difference: enhancement vs restoration. Enhancement: subjective improvement; restoration: mathematical reversal of degradation.

Four × 2.5-mark slightly longer questions

  • Explain pipeline steps (short): Acquisition → Enhancement/Restoration → Morphology → Segmentation → Recognition (one line each).
  • Explain f(x,y)=i(x,y)r(x,y). Illumination × reflectance; illumination varies with lighting, reflectance with object properties; product gives observed intensity.
  • Sampling & quantization example: 512×512, 8-bit → M=512,N=512,k=8,L=256; storing as matrix A of size 512×512.
  • List components of processing system and role of sensor and display (two lines).

Two × 5-mark long questions

  • Discuss sampling vs quantization effects and artifacts: Explain how reducing M (spatial) causes aliasing/jaggies and reducing k (intensity) causes contouring; give remedies (oversample, anti-alias filters, increase bit depth, dithering).
  • Explain full pipeline with emphasis on segmentation: For each pipeline step give purpose, common methods (filters for enhancement, deconvolution for restoration, morphological ops, threshold/edge/region growing for segmentation) and why segmentation is hardest (object variability, occlusion) and critical for recognition.
