1. What is a Histogram?

Think of a digital image as a bag of pixels, each carrying a brightness number (intensity). A histogram answers one question: how many pixels have each brightness level? You sort all pixels by their intensity and count how many fall into each level. That count, plotted as a bar chart, is the histogram.

1.1 Unnormalized Histogram

The raw count of pixels at each intensity level:

$$h(r_k) = n_k$$
  • $r_k$: the $k$-th intensity level (e.g., 0, 1, 2, ..., $L-1$)
  • $n_k$: number of pixels in the image that have intensity $r_k$

Each intensity level occupies one bin, a slot on the intensity scale that holds the pixel count for that level. An 8-bit image has 256 bins (0 through 255). A 3-bit image has 8 bins (0 through 7).

1.2 Normalized Histogram

Divide each count by the total number of pixels in the image:

$$p(r_k) = \frac{n_k}{M \times N}$$
  • $M \times N$: total pixels (image height $\times$ width)
  • $p(r_k)$: fraction of all pixels at intensity $r_k$, a value between 0 and 1

Why normalize? Because raw counts depend on image size. A 100×100 image and a 1000×1000 image cannot be compared by raw counts. Normalizing makes the histogram size-independent. It also turns the histogram into a valid probability distribution, every $p(r_k) \geq 0$ and all values sum to exactly 1.

2. Histogram as a Contrast Indicator

The shape of a histogram tells you the contrast quality of the image instantly, without even looking at the image itself:

  • Narrow histogram (pixels clustered in a small intensity band) → low contrast. All pixels are nearly the same brightness. The image looks flat, like a foggy photograph where everything blends into a uniform gray.
  • Broad histogram (pixels spread across the full $[0, L-1]$ range) → high contrast. There is a wide gap between the darkest and brightest pixels. The image looks crisp with clear distinction between features.

The reason is simple: if every pixel lies between intensity 100 and 150, the maximum brightness difference anywhere in the image is only 50 levels, your eye can barely distinguish features. If pixels span 0 to 255, the difference is 255 levels, maximum possible visual separation.

All normalized histogram values sum to 1, because every pixel belongs to exactly one bin. This is a direct consequence of it being a probability distribution.

3. Histogram Equalization

This is the most important histogram technique. It takes any image, dark, bright, low-contrast, and automatically improves its contrast by redistributing pixel intensities across the full available range.

3.1 The Core Idea

A low-contrast image has a narrow histogram. The fix: spread the histogram so it covers the entire intensity range. The ideal target is a perfectly uniform (flat) histogram, where every intensity level has exactly the same number of pixels. This gives maximum contrast because no brightness region is favored over another.

Analogy: 40 students all scored between 70–80%. You cannot tell who is better. Now "curve" the grades so they spread evenly from 0–100%. Suddenly, performance differences become obvious. That curving operation is histogram equalization.

3.2 Conditions for a Valid Transformation

Before deriving the formula, any intensity transformation $s = T(r)$ must satisfy two conditions:

  • Condition (a), Monotonically increasing: If $r_1 < r_2$, then $T(r_1) \leq T(r_2)$. A brighter input must produce a brighter (or equal) output. This prevents intensity reversal artifacts, situations where a dark pixel gets mapped brighter than a previously brighter pixel, creating visual nonsense.
  • Condition (b), Output within range: $0 \leq T(r) \leq L-1$. The output cannot exceed the valid intensity range of the image.

A stronger variant, Condition (a'), Strictly monotonic, requires $T(r_1) < T(r_2)$ (never equal). This makes the mapping one-to-one and invertible, you can recover the original from the output. Condition (a') is not always achievable in discrete images.

3.3 The Continuous Formula

In the continuous (theoretical) case, the equalization transform is:

$$s = T(r) = (L-1) \int_0^r p_r(w) \, dw$$

In words: the new intensity $s$ equals $(L-1)$ multiplied by the Cumulative Distribution Function (CDF) of the input image, evaluated at $r$.

3.3.1 What is the CDF?

The CDF at intensity $r$ is the running total of probabilities from 0 up to $r$. It answers: what fraction of all pixels have intensity $\leq r$? Mathematically, it is the integral (area under the curve) of the probability density function (PDF) from 0 to $r$.

3.3.2 Why Does the CDF Work?
  • Condition (a) satisfied: The PDF $p_r(w) \geq 0$ everywhere. Integrating a non-negative function means the result can only increase or stay flat, never decrease. So the CDF is monotonically increasing. ✓
  • Condition (b) satisfied: At $r = 0$, the integral is 0, so $s = 0$. At $r = L-1$, the integral of the entire PDF equals 1, so $s = (L-1) \times 1 = L-1$. Output range is exactly $[0, L-1]$. ✓
  • The output distribution is uniform: In the continuous case, it is mathematically provable that the output PDF $p_s(s)$ is always $\frac{1}{L-1}$, a constant. This means the CDF-based mapping always produces a perfectly flat histogram, regardless of the input shape.

One-liner to remember: Histogram equalization = scale the CDF by $(L-1)$.

3.4 The Discrete Formula (The One You Compute)

Real images are digital, intensities are integers, not continuous values. Replace the integral with a summation:

$$s_k = T(r_k) = (L-1) \sum_{j=0}^{k} p_r(r_j) = \frac{L-1}{MN} \sum_{j=0}^{k} n_j$$

In words: for each intensity level $k$, the new value $s_k$ equals $(L-1)$ times the cumulative sum of the normalized histogram up to and including bin $k$, rounded to the nearest integer.

3.4.1 Step-by-Step Algorithm

Given an image of size $M \times N$ with $L$ intensity levels:

  • Step 1: List all intensity levels $r_k$ and count their pixel occurrences $n_k$.
  • Step 2: Compute normalized histogram: $p_r(r_k) = n_k / (M \times N)$ for each $k$.
  • Step 3: Compute the cumulative sum $\text{CDF}(k) = \sum_{j=0}^{k} p_r(r_j)$ for each $k$.
  • Step 4: Multiply each CDF value by $(L-1)$: $s_k = (L-1) \times \text{CDF}(k)$.
  • Step 5: Round each $s_k$ to the nearest integer. This is the new intensity for all pixels that originally had intensity $r_k$.
# Histogram Equalization in Python
import numpy as np

def histogram_equalize(image, L=256):
    MN = image.size
    hist, _ = np.histogram(image.flatten(), bins=L, range=[0, L])
    pdf = hist / MN
    cdf = np.cumsum(pdf)
    mapping = np.round((L - 1) * cdf).astype(np.uint8)
    return mapping[image]

3.6 Practical Realities of Discrete Equalization

  • Perfect uniformity is rare: The continuous theory guarantees a flat output. The discrete case involves rounding, so perfect flatness is almost never achieved.
  • No mathematical proof for discrete case: Unlike the continuous case, you cannot formally prove that discrete histogram equalization yields a uniform histogram.
  • Fully automatic: Zero parameters needed. Feed the image, get the result. This is the single biggest practical advantage, no tuning, no user expertise required.
  • Invertible: The mapping $r_k \to s_k$ can be reversed ($r_k = T^{-1}(s_k)$) if condition (a') holds, i.e., if no two input levels map to the same output level. When they do (as in the example above, where both $r=3$ and $r=4$ map to 6), the inverse is ambiguous.

4. Histogram Matching (Specification)

4.1 Why Matching Exists

Histogram equalization has one fundamental limitation: it always targets a uniform (flat) histogram. You have zero control over the output distribution shape. But sometimes you need a specific output histogram, perhaps to match a reference image's look, or to achieve a particular artistic distribution. Histogram matching (also called histogram specification) solves this by letting you specify any desired output histogram shape.

In one line: Equalization forces a flat output. Matching lets you choose any output shape.

4.2 The Continuous Theory

The trick uses equalization as a stepping stone, a universal middleman. The idea is beautifully simple:

  • Step 1: Equalize the input image. The mapping $s = T(r) = (L-1) \int_0^r p_r(w) \, dw$ takes the input $r$ to a uniform variable $s$.
  • Step 2: Define the desired output. Compute $G(z) = (L-1) \int_0^z p_z(w) \, dw$, the CDF-based mapping that would equalize the desired distribution. This takes the desired output $z$ to the same uniform variable $s$.
  • Step 3: Since both $T(r)$ and $G(z)$ map to the same uniform $s$, chain them by inverting $G$:
$$z = G^{-1}(s) = G^{-1}\big(T(r)\big)$$

Analogy: two currencies both convert to USD. Rupee → USD → Euro. The uniform histogram plays the role of USD, a common intermediate. Flow: input $r$ → equalize → uniform $s$ → invert desired CDF → target $z$.

4.3 The Discrete Algorithm (4 Steps)

This is the exam-ready procedure. Given: input image with histogram $p_r(r_k)$, and a specified (desired) histogram $p_z(z_q)$.

Step 1: Equalize the Input

Compute $s_k = \text{round}\left[(L-1) \sum_{j=0}^{k} p_r(r_j)\right]$ for each intensity level $k$. This is identical to the histogram equalization algorithm from Section 3.4.

Step 2: Compute G from the Specified Histogram

Using the desired histogram $p_z(z_q)$, compute:

$$G(z_q) = \text{round}\left[(L-1) \sum_{i=0}^{q} p_z(z_i)\right]$$

for each $q = 0, 1, ..., L-1$. Store these rounded values in a lookup table.

Step 3: Match s to z

For each equalized value $s_k$, find the value $z_q$ whose $G(z_q)$ is closest to $s_k$. If multiple $z_q$ values give the same closest match, choose the smallest $z_q$ by convention.

Step 4: Map Pixels

For every pixel in the original image with intensity $r_k$: look up its equalized value $s_k$ (from Step 1), then look up the matched value $z_q$ (from Step 3). Write $z_q$ to the output image.

Shortcut: You never need to physically create the equalized image. Compute the $r \to s$ mapping and the $s \to z$ mapping, then chain them directly: $r \to z$. The intermediate equalized image is purely computational, it never needs to exist as an actual image.

5. Global vs. Local Histogram Processing

5.1 The Problem with Global Methods

Both histogram equalization and histogram matching are global methods, they compute a single histogram from the entire image and apply one transformation to all pixels. This creates a blind spot: if a small region of the image contains hidden details (e.g., faint letters in a dark corner), those few pixels are a tiny fraction of the total. Their contribution to the global histogram is negligible, so the global transformation barely affects them. The details remain invisible.

5.2 Local Histogram Processing

Define a small sliding window (neighborhood), e.g., $3 \times 3$. Place its center on a pixel. Compute the histogram of only the pixels inside this window. Apply equalization (or matching) using this local histogram. Update only the center pixel. Slide the window to the next pixel and repeat across the entire image via raster scanning.

Because the histogram now reflects only the immediate neighborhood, even faint details that are locally distinct get stretched to full contrast. A $3 \times 3$ local equalization can reveal hidden shapes (letters, edges, symbols) that global equalization missed entirely.

5.3 Efficiency: Incremental Histogram Update

When the window slides one pixel to the right, only one column of pixels leaves the window and one column enters. You do not recompute the histogram from scratch, subtract the counts of the departing column and add the counts of the arriving column. This incremental update makes local processing computationally feasible even for large images.
