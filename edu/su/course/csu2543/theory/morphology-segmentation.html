1. Morphological Processing: Preliminaries

Everything covered so far, intensity transforms, filtering, frequency-domain operations, works at the pixel level. Morphological processing moves one layer higher: it extracts and manipulates shape-level information from binary images. It answers questions like: "Is this a thin line or a thick blob? Does this object have holes? Are these two regions connected?"

The mathematical foundation is set theory. A binary image is treated as a set of foreground pixel coordinates. Morphological operations probe this set by systematically moving a small, predefined shape across it.

1.1 The Structuring Element

The structuring element (SE) is a small binary shape, a template, that defines how the image is probed. Common shapes: a $3 \times 3$ square, a cross, a disk, a line. Think of it as a cookie cutter: its shape determines which pixels are examined at each position, and its size controls the scale of the operation.

The SE has a designated origin (center point). The origin is placed on each foreground pixel, and the relationship between the SE and the underlying image determines the output.

2. Erosion

Erosion shrinks (thins) objects in a binary image. Conceptually: place the structuring element's origin on every foreground pixel. If the entire SE fits within the foreground at that position, the pixel survives. If any part of the SE falls outside the foreground, the pixel is removed.

2.1 Effects

  • Objects shrink, boundaries recede inward by an amount proportional to the SE size.
  • Small objects disappear entirely, if an object is smaller than the SE, it cannot contain the SE anywhere, so it is completely erased.
  • Thin connections break, narrow bridges between objects get eroded away, separating touching objects.

2.2 Primary Use

Removing small irrelevant details and noise from a binary image. Noise in a binary image typically appears as isolated small white pixels or tiny clusters, all smaller than the SE, so erosion eliminates them.

3. Dilation

Dilation is the dual (opposite) of erosion. It grows (thickens) objects. Conceptually: place the SE's origin on every foreground pixel and "stamp" the entire SE shape onto the output. Every pixel touched by the SE at any foreground position becomes foreground.

3.1 Effects

  • Objects expand, boundaries push outward.
  • Small gaps and holes fill in, if a gap is smaller than the SE, dilation bridges it.
  • Nearby objects merge, close objects grow until they touch.

3.2 Primary Use

Bridging gaps in broken boundaries and filling small holes within objects. If an edge detector produced a nearly-closed contour with a few pixel-wide breaks, dilation can close them.

4. Opening and Closing

Erosion and dilation are rarely used alone, they change the object's size. By combining them in specific sequences, we get operations that clean shape features without significantly altering overall size.

4.1 Opening = Erosion → Dilation

First erode (shrink), then dilate (grow back). The erosion removes small noise and thin protrusions. The subsequent dilation restores the object approximately to its original size, but the removed features don't come back, because they were completely eliminated by the erosion.

Effects: Smooths object contours from the outside. Breaks narrow isthmuses (thin connections). Eliminates thin protrusions and spurs. Removes noise smaller than the SE.

4.2 Closing = Dilation → Erosion

First dilate (grow), then erode (shrink back). The dilation fills small holes, gaps, and narrow breaks. The subsequent erosion restores the object approximately to its original size, but the filled features stay filled.

Effects: Smooths object contours from the inside. Fuses narrow breaks and gaps. Fills small holes. Connects nearby components.

4.3 Morphological Noise Filtering

A common pipeline: Opening followed by Closing. Opening removes external noise (small bright spots on dark background). Closing removes internal noise (small dark holes inside objects). Together, they clean both types without significantly distorting the original shapes.

# Morphological operations
import cv2
import numpy as np

# Structuring element (3x3 square)
SE = np.ones((3, 3), dtype=np.uint8)

eroded  = cv2.erode(binary_image, SE, iterations=1)
dilated = cv2.dilate(binary_image, SE, iterations=1)
opened  = cv2.morphologyEx(binary_image, cv2.MORPH_OPEN, SE)
closed  = cv2.morphologyEx(binary_image, cv2.MORPH_CLOSE, SE)

5. Image Segmentation Fundamentals

Segmentation is the process of dividing an image into meaningful regions or objects. Everything before this, enhancement, filtering, restoration, was about making the image look better or cleaner. Segmentation is about understanding what is in the image: separating the foreground objects from the background, identifying individual regions, locating boundaries.

Accurate segmentation is critical. All subsequent analysis (object recognition, measurement, classification) depends on it. Over-segment (too many regions) → noise. Under-segment (too few) → lost objects.

5.1 Two Fundamental Approaches

  • Discontinuity-based, detect abrupt changes in intensity (edges). The assumption: object boundaries correspond to intensity discontinuities. Tools: gradient operators (Sobel, Roberts), Laplacian, Canny edge detector.
  • Similarity-based, group pixels that share similar properties (intensity, color, texture). The assumption: pixels within the same object have similar values. Tools: thresholding, region growing, clustering.

6. Edge Models and Gradients

Edges are the most fundamental features for segmentation. An edge is a set of connected pixels that lie on the boundary between two regions with different intensities.

6.1 Edge Models

  • Step edge: An idealized, instantaneous transition between two intensity levels, the change occurs over exactly 1 pixel. Real images never produce perfect step edges.
  • Ramp edge: A realistic model where the transition is gradual, spread over several pixels due to optical blur, sampling, and noise. The edge has a finite width.
  • Roof edge: A thin line structure, intensity rises from the background, remains elevated for a few pixels, then falls back. Example: a thin wire against a uniform background.

6.2 Detecting Edges with Derivatives

  • First-order derivative (gradient), peaks at the midpoint of a ramp edge. Gives strong response along the entire edge transition. Used for: Sobel, Roberts operators.
  • Second-order derivative (Laplacian), crosses zero at the midpoint of a ramp edge. Gives a double response (positive and negative) at the onset and end of the transition. The zero-crossing precisely locates the edge center.

These are the same derivative operators from spatial filtering (Laplacian, Sobel), now applied specifically for segmentation rather than enhancement.

7. Image Thresholding

The simplest and most direct segmentation method. Given a threshold value $T$, classify every pixel:

$$g(x,y) = \begin{cases} 1 & \text{if } f(x,y) \geq T \quad (\text{object}) \\ 0 & \text{if } f(x,y) < T \quad (\text{background}) \end{cases}$$

The result is a binary image, pixels are either object (1/white) or background (0/black). Thresholding works best when the image contains light objects on a dark background (or vice versa) and the histogram shows two distinct peaks (bimodal).

7.1 Global Thresholding

A single constant threshold $T$ is used for the entire image. Works well only under controlled, uniform illumination where the histogram has clear, well-separated bimodal peaks. If illumination is uneven, a single $T$ cannot correctly separate objects in both the bright and dark regions of the image.

7.2 Variable (Local/Adaptive) Thresholding

The threshold varies across the image. The image is divided into sub-regions (or a sliding window is used), and a different threshold is computed for each region based on local statistics. This compensates for uneven illumination and shading, the same object may require $T = 100$ in a dark region and $T = 180$ in a bright region.

7.3 Otsu's Method (Optimum Global Thresholding)

The most widely used automatic thresholding algorithm. It selects the threshold $T$ that maximizes the between-class variance, the variance between the "object" class and the "background" class.

7.3.1 Intuition

A good threshold should make the two groups as different from each other as possible. The between-class variance measures how far apart the mean intensities of the two groups are, weighted by how many pixels fall in each. Maximizing this variance means finding the $T$ that most cleanly separates the two populations.

7.3.2 Algorithm
  • Step 1: Compute the normalized histogram $p(i)$ for all intensity levels $i = 0, 1, \ldots, L-1$.
  • Step 2: For every possible threshold $T$ from 0 to $L-1$:
    • Compute the probability of each class: $\omega_1(T) = \sum_{i=0}^{T} p(i)$ and $\omega_2(T) = 1 - \omega_1(T)$.
    • Compute the mean of each class: $\mu_1(T)$ and $\mu_2(T)$.
    • Compute the between-class variance: $\sigma_B^2(T) = \omega_1(T) \cdot \omega_2(T) \cdot [\mu_1(T) - \mu_2(T)]^2$.
  • Step 3: The optimal threshold $T^*$ is the one that maximizes $\sigma_B^2(T)$.

Key advantage: Fully automatic, no parameters, no manual tuning. Robust standard for automated systems.

# Otsu's thresholding
import cv2

# Automatic threshold selection
T, binary = cv2.threshold(gray_image, 0, 255,
                          cv2.THRESH_BINARY + cv2.THRESH_OTSU)
print(f"Otsu's optimal threshold: {T}")

8. Region-Based Segmentation: Region Growing

Edge-based methods find boundaries. Region-based methods find areas. Region growing takes the similarity approach: instead of looking for where values change abruptly, it groups pixels that are similar to each other.

8.1 Algorithm

  • Step 1, Choose seed points: Manually or automatically select one or more starting pixels known to be inside the region of interest.
  • Step 2, Define a similarity criterion: A rule for deciding whether a neighboring pixel "belongs" to the same region. Common criteria: intensity difference $|f(x,y) - \text{mean}_{\text{region}}| < \text{threshold}$, or color similarity, or texture similarity.
  • Step 3, Grow: For each seed, examine its neighbors. If a neighbor satisfies the similarity criterion, add it to the region and make it a new candidate for expansion. Repeat until no more pixels can be added.
  • Step 4, Stop: Growth stops when no neighboring pixel satisfies the criterion. The final set of connected pixels is the segmented region.

Strength: Very effective when you have prior knowledge about the object, you know approximately where it is (seed point) and what it looks like (similarity rule). Produces connected regions by construction.

Weakness: Sensitive to seed selection and the similarity threshold. Too loose a criterion → regions leak into the background. Too strict → regions fragment.
