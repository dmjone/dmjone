Intensity Transformations, Digital Image Processing

Intensity Transformations

2026, April 12

1. How Images Are Processed: Two Domains

A digital image is a 2D grid of numbers. Each number is a pixel's intensity (brightness). To enhance this image, you manipulate these numbers. There are exactly two places where you can do this:

  • Spatial Domain, work directly on the pixel values themselves. This is like editing a spreadsheet cell by cell.
  • Frequency Domain, convert the entire image into a different mathematical representation (using the Fourier Transform), edit there, then convert back. This document does not cover this.

Within the spatial domain, there are two distinct methods:

  • Intensity Transformation, each pixel is modified independently, based only on its own value. No neighbor is consulted. Used for image enhancement (brightness, contrast).
  • Spatial Filtering, each pixel is modified based on itself and its neighbors. Used for denoising (e.g., averaging a 3×3 block of pixels to smooth noise).
This entire document covers only intensity transformations, the single-pixel operations.

2. The Core Equation of Intensity Transformation

Every intensity transformation reduces to one line:

$$s = T(r)$$
  • $r$, input pixel intensity (what you read from the image)
  • $T$, the transformation function (a rule that maps old values to new values)
  • $s$, output pixel intensity (what you write to the new image)

2.1 Key Property: Location-Independent

The transformation depends only on the pixel's intensity value, not on where it sits in the image. A pixel with value 100 at position $(0,0)$ gets exactly the same output as a pixel with value 100 at position $(200,300)$. The function $T$ does not take coordinates as input, only the brightness value.

2.2 Execution: Raster Scanning

The image is processed by visiting every pixel in raster order: start at the top-left corner, move left-to-right across the row, then drop to the next row and repeat, until you reach the bottom-right corner. At each pixel: read $r$, compute $T(r)$, write $s$, move on.

2.3 Contrast with Spatial Filtering

Spatial filtering also uses raster scanning, but its formula is different:

$$g(x,y) = T[f(x,y)]$$

Here, $T$ operates on a neighborhood of pixels (typically a square, e.g., 3×3) centered at $(x,y)$. The output depends on the pixel and its surrounding neighbors. Example: replacing each pixel with the average of its 3×3 neighborhood smooths the image.

Remember the difference: Intensity transform = one pixel in, one pixel out. Spatial filter = a block of pixels in, one pixel out.

3. Look-Up Tables and the Master Graph

3.1 Why a Look-Up Table (LUT)?

In a digital image, $r$ and $s$ are discrete integers in the range $[0, L-1]$, where $L$ is the total number of intensity levels (typically $L = 256$ for an 8-bit image, giving values 0 through 255). Since there are only 256 possible inputs, you can precompute $T(r)$ for every possible $r$ and store the results in an array of size 256. This array is the look-up table.

At runtime, instead of computing $T(r)$ through a formula for each of millions of pixels, you just index into the array: output = LUT[r]. This is an $O(1)$ operation per pixel, extremely fast.

# Building a LUT for a generic transform T
LUT = [T(r) for r in range(256)]

# Applying it to every pixel
for each pixel (x, y):
    r = image[x][y]
    output[x][y] = LUT[r]

3.2 The Master Graph: All Transforms at a Glance

Plot input intensity $r$ on the x-axis and output intensity $s$ on the y-axis. Every transform is a curve on this graph. All valid curves pass through two anchor points: $(0, 0)$ and $(L-1, L-1)$.

  • The 45° diagonal ($s = r$) is the identity line, no change to the image.
  • Curves above the identity → output is brighter than input for that range → brightens the image.
  • Curves below the identity → output is darker than input → darkens the image.

All transforms in this document (negative, log, power law, etc.) are different curves on this same graph. Knowing whether a curve sits above or below the identity line tells you instantly whether it brightens or darkens.

4. Image Inversion (Negative)

$$s = (L - 1) - r$$

This is the simplest intensity transformation. It flips the entire intensity scale: black (0) becomes white ($L-1$), white becomes black, and every shade in between is mirrored. A pixel with value 200 in an 8-bit image becomes $255 - 200 = 55$.

4.1 When to Use

When the image is dominated by dark (black) regions and the features of interest are faint details hidden in those dark areas. Inverting turns the dark background bright and makes the subtle details visible as dark features on a light background.

4.2 Classic Application

Mammography: breast tissue appears mostly dark on X-ray films. Tumors are subtle, slightly brighter spots. After applying the negative transform, these features become much easier for a radiologist to spot.

# Image Negative in Python (NumPy)
import numpy as np
output = 255 - image   # for 8-bit image, L=256

5. Log Transformation

$$s = c \cdot \log(1 + r)$$
  • $c$, a scaling constant, chosen to map the output range to $[0, L-1]$
  • $1 + r$, the "+1" prevents $\log(0)$, which is undefined

5.1 What It Does Intuitively

Think of $\log$ as a magnifying glass for the dark end of the intensity scale:

  • Dark pixels (small $r$): The log function rises steeply here. Small differences in dark values get stretched apart in the output → more visible detail in shadows.
  • Bright pixels (large $r$): The log function flattens out. Large differences in bright values get compressed together → less distinction in highlights.

The net effect: dark areas get enhanced, bright areas get compressed. The curve sits above the identity line → overall brightening.

5.2 Killer Application: Fourier Spectrum Display

The Fourier transform of an image produces values ranging from 0 to potentially millions. If you try to display this directly on a screen (which can show only 0–255), everything looks black except for one or two white dots. Applying the log transform compresses this enormous dynamic range into the 0–255 displayable range, making the frequency content visible.

# Log transform
import numpy as np
c = 255 / np.log(1 + np.max(image))
output = c * np.log(1 + image.astype(np.float64))
output = np.uint8(np.clip(output, 0, 255))

6. Power Law (Gamma) Transformation

$$s = c \cdot r^{\gamma}$$

Here $r$ and $s$ are normalized to $[0, 1]$ (divide by $L-1$ before applying, multiply back after). $c$ is usually 1. The single parameter $\gamma$ (gamma) controls the entire behavior:

6.1 Three Regimes of Gamma

Condition Curve Position Effect
$\gamma < 1$ Above identity Brightens dark areas (expands low intensities, compresses high)
$\gamma = 1, c = 1$ On identity No change
$\gamma > 1$ Below identity Darkens image (compresses low intensities, expands high)

Memory trick: $\gamma < 1$ lifts dark values up (brightening). $\gamma > 1$ pushes them down (darkening). The further $\gamma$ is from 1, the stronger the effect.

This is the most versatile intensity transform because a single tunable knob ($\gamma$) can handle both dark and bright images.

6.2 Contrast Enhancement: Practical Usage

6.2.1 Fixing a Dark Image

Use $\gamma < 1$. Example values: $\gamma = 0.6, 0.4, 0.3$ with $c = 1$. Smaller $\gamma$ → stronger brightening. A dark spine X-ray becomes readable at $\gamma = 0.3$.

6.2.2 Fixing a Bright (Washed-Out) Image

Use $\gamma > 1$. Example values: $\gamma = 3.0, 4.0, 5.0$ with $c = 1$. Larger $\gamma$ → stronger darkening, recovering lost detail in bright regions. An overexposed aerial photograph regains detail at $\gamma = 5.0$.

# Gamma correction
import numpy as np
gamma = 0.4   # < 1 for dark images, > 1 for bright images
c = 1
# Normalize to [0,1], apply gamma, scale back
normalized = image / 255.0
output = c * np.power(normalized, gamma)
output = np.uint8(np.clip(output * 255, 0, 255))

7. Gamma Correction for Display Devices

7.1 The Problem

CRT monitors (and many display/capture devices) do not display intensities linearly. They internally apply a power law: $s = r^{\gamma}$ where $\gamma \approx 1.8$ to $2.5$. So when you send a mid-gray pixel (intensity = 0.5) to a CRT with $\gamma = 2.5$, it displays:

$$s = 0.5^{2.5} \approx 0.18$$

That is far darker than the intended 0.5. Every shade gets distorted, the output is too dark.

7.2 The Fix: Pre-Correction

Before sending the image to the device, apply the inverse gamma:

$$s_{\text{pre}} = r^{1/\gamma}$$

When the device then applies its internal $\gamma$, the two cascade:

$$\text{Device output} = \left(r^{1/\gamma}\right)^{\gamma} = r^{(1/\gamma) \cdot \gamma} = r^1 = r$$

The original intensity is perfectly restored. They cancel out.

Concrete example: CRT has $\gamma = 2.5$. You pre-correct with $\gamma = 1/2.5 = 0.4$. A pixel at 0.5 gets pre-corrected to $0.5^{0.4} \approx 0.76$. The CRT then applies $0.76^{2.5} \approx 0.5$. Original restored.

8. Summary: Choosing the Right Transform

Transform Formula Curve Shape Use When
Identity $s = r$ 45° diagonal Reference only (no change)
Negative $s = (L-1) - r$ Declining line Dark-dominant images; flip contrast
Log / $n$th root $s = c \cdot \log(1+r)$ Convex (above identity) Image too dark; expand dark regions
Power law, $\gamma > 1$ $s = c \cdot r^{\gamma}$ Concave (below identity) Image too bright; expand bright regions
Power law, $\gamma < 1$ $s = c \cdot r^{\gamma}$ Convex (above identity) Image too dark; brighten
One rule to remember them all: Curve above identity = brightens. Curve below identity = darkens.

9. Contrast Stretching

9.1 The Problem It Solves

Some images use only a narrow band of the available intensity range, e.g., all pixels lie between 80 and 180 in a 0–255 scale. The image looks flat, with low contrast. Contrast stretching expands this narrow range to fill the entire $[0, L-1]$ range, making differences more visible.

9.2 How It Works

Define a piecewise linear function using two control points $(r_1, s_1)$ and $(r_2, s_2)$. The segment between these points has a steep slope (stretches that intensity range), while the segments outside are flatter (compresses those ranges).

9.3 Monotonicity Constraint

The transformation function must be monotonically increasing: if $r_1 \leq r_2$, then $s_1 \leq s_2$. This prevents intensity reversals, where a darker pixel becomes brighter than a previously brighter pixel, which creates visual artifacts. However, the function need not be strictly increasing: flat segments (where a range of inputs maps to the same output) are allowed.

9.4 Special Case: Thresholding

If the piecewise function becomes a step function (a single vertical jump at one threshold), the output is binary: every pixel becomes either 0 (black) or $L-1$ (white). This is thresholding, the simplest form of image segmentation.

10. Intensity-Level Slicing

10.1 The Idea

You care only about pixels whose intensity falls within a specific range $[A, B]$, for example, blood vessels in a medical image that appear at a particular brightness. You want to highlight these and suppress or preserve everything else.

10.2 Two Approaches

10.2.1 Binary Slicing (Without Background)

Pixels in $[A, B]$ → set to a high value (e.g., white). All other pixels → set to 0 (black). Result: only the features of interest are visible; everything else disappears. Think of it as cutting out a brightness band from the image.

10.2.2 Background-Preserving Slicing

Pixels in $[A, B]$ → set to a high value. All other pixels → keep their original intensity. Result: features are highlighted (bright), but the rest of the image remains visible for context.

# Intensity-Level Slicing
A, B = 100, 180  # range of interest

# Binary slicing
binary = np.where((image >= A) & (image <= B), 255, 0)

# Background-preserving
preserved = np.where((image >= A) & (image <= B), 255, image)

10.3 Application Example

Aortic angiogram near the kidney: blood vessels occupy a specific intensity range. Binary slicing isolates vessels as bright on black. Background-preserving slicing highlights vessels while retaining surrounding anatomical structures.

11. Bit-Plane Slicing

11.1 Core Concept

In an 8-bit image, each pixel value is stored as 8 binary digits. For example, pixel value 167 in binary is 10100111. You can decompose the entire image into 8 separate binary images, one per bit position. Each binary image is a bit-plane.

11.2 Significance of Each Plane

  • Bit-plane 8 (MSB, Most Significant Bit): Contributes $2^7 = 128$ to the pixel value. This plane captures the coarse structure of the image. Visually, it looks most like the original.
  • Bit-plane 7: Contributes $2^6 = 64$. Still structurally meaningful.
  • Bit-planes 6, 5: Finer detail, less dominant.
  • Bit-plane 1 (LSB, Least Significant Bit): Contributes only $2^0 = 1$. Appears as near-random noise. Almost no visual meaning.
11.2.1 Extracting a Bit-Plane
# Extract bit-plane n (1 = LSB, 8 = MSB) from image
def extract_bit_plane(image, n):
    # Shift right by (n-1), then mask the lowest bit
    return ((image >> (n - 1)) & 1) * 255

11.3 Application: Image Compression

Since the lower bit-planes carry almost no recognizable information, you can discard them and reconstruct the image from only the top few planes. This is lossy compression:

Planes Used Bits per Pixel Compression Quality
8 + 7 2 75% Recognizable structure
8 + 7 + 6 3 62.5% Good approximation
8 + 7 + 6 + 5 4 50% Near-original quality 
# Reconstruct image from top-k bit-planes
def reconstruct(image, top_k):
    result = np.zeros_like(image)
    for n in range(8, 8 - top_k, -1):
        result += ((image >> (n - 1)) & 1) << (n - 1)
    return result
Exam definition: Bit-plane slicing decomposes an image into binary images, one per bit position, revealing each bit's relative contribution to the overall visual appearance.
