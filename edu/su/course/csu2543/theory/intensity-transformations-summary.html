Intensity Transformations Summary

2026, April 12
View Main Article

1. What Problem Are We Solving?

An image is a grid of numbers (pixel intensities). Enhancement = transforming these numbers.

There are two domains:

  • Spatial Domain: Directly modify pixel values
  • Frequency Domain: Modify after transformation (not covered)

Within spatial domain:

  • Intensity Transformation: One pixel → one output (independent)
  • Spatial Filtering: Pixel + neighbors → output (dependent)

Key Insight: Intensity transformation = function on value, not position.

2. Core Model: One Equation Controls Everything

The entire topic reduces to:

$$ s = T(r) $$

  • r: input intensity
  • T: transformation function
  • s: output intensity

2.1 Critical Property

Transformation is location-independent.

Same input value → same output, regardless of pixel position.

2.2 Execution Model

Raster Scan: left → right, top → bottom

  • Read r
  • Apply T(r)
  • Write s

3. Optimization: Look-Up Table (LUT)

Since intensity values are discrete (0–255), precompute:

LUT = [T(r) for r in range(256)]
output[x][y] = LUT[input[x][y]]

Concept: Replace computation with memory → O(1) per pixel.

4. Master Graph Thinking (Most Important)

Plot: x-axis = input (r), y-axis = output (s)

  • Identity line: $s=r$
  • Above line → brightens
  • Below line → darkens

Everything reduces to curve shape.

5. Image Negative (Inversion)

$$ s = (L - 1) - r $$

Effect: flips intensity

  • Black → White
  • Bright → Dark

Use: reveal details in dark-dominant images

Implementation
output = 255 - image

6. Log Transformation

$$ s = c \cdot \log(1 + r) $$

6.1 Behavior

  • Dark values → expanded (detail increases)
  • Bright values → compressed

Concept: Magnifies shadow information

6.2 Key Application

Fourier spectrum visualization (compress huge ranges)

Implementation
c = 255 / np.log(1 + np.max(image))
output = c * np.log(1 + image)

7. Power Law (Gamma Transformation)

$$ s = c \cdot r^\gamma $$

7.1 Three Regimes

  • γ < 1: Brightens dark regions
  • γ = 1: No change
  • γ > 1: Darkens image

Core Insight: One parameter controls entire behavior

7.2 Practical Usage

  • Dark image → γ = 0.3–0.6
  • Bright image → γ = 3–5
Implementation
normalized = image / 255.0
output = np.power(normalized, gamma) * 255

8. Gamma Correction (Device Compensation)

Devices apply:

$$ s = r^\gamma $$

Problem: Image appears darker

8.1 Solution

Pre-correct using:

$$ s = r^{1/\gamma} $$

Why it works:

$$ (r^{1/\gamma})^\gamma = r $$

Concept: Cancel device distortion before display

9. Contrast Stretching

Problem: Intensities occupy narrow range

9.1 Solution

Expand to full range using piecewise linear function

9.2 Constraint

Must be:

$$ r_1 \leq r_2 \Rightarrow s_1 \leq s_2 $$

Reason: Prevent intensity reversal artifacts

9.3 Special Case

Step function → Thresholding

10. Intensity Level Slicing

Goal: Highlight specific intensity range [A,B]

10.1 Methods

  • Binary: keep only [A,B], rest → 0
  • Preserving: highlight [A,B], keep others
Implementation
binary = np.where((img>=A)&(img<=B),255,0)
preserve = np.where((img>=A)&(img<=B),255,img)

11. Bit-Plane Slicing

11.1 Concept

Each pixel = 8-bit binary → split into 8 images

11.2 Importance

  • MSB → structure
  • LSB → noise

11.3 Extraction

def extract_bit_plane(img, n):
    return ((img >> (n-1)) & 1) * 255

11.4 Compression Insight

  • Top 2 bits → recognizable image
  • Lower bits → negligible information

12. Decision Engine (Exam Gold)

  • Too dark? → Log or γ < 1
  • Too bright? → γ > 1
  • Need binary segmentation? → Thresholding
  • Need feature isolation? → Intensity slicing
  • Need compression insight? → Bit-plane slicing
  • Need fast computation? → LUT

Master Rule: Curve above identity → brightens, below → darkens
