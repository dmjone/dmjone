Summary: Digital Image Processing

2026, March 8

1. Digital Image

  • 2D function f(x, y): x, y = position, f = intensity
  • All three finite, discrete → digital image
  • Pixel: smallest unit. Has location (x, y) and value (brightness)
  • Size M × N = M·N pixels. M rows, N columns

2. Processing Levels

LevelInput → OutputPurpose
LowImage → ImagePixel cleanup: noise reduction, contrast, sharpening
MidImage → AttributesFact extraction: edges, segmentation, classification
HighAttributes → MeaningScene understanding, autonomous navigation

Low = mechanical. High = intelligent. Abstraction increases at each level.

4.2 Non-EM Sources

  • Ultrasound: Sound reflects off tissue boundaries (20 kHz–200 MHz)
  • Electron microscopy: Nanometer-scale imaging via electron beams
  • Synthetic: Computer-generated (3D models, fractals), no sensor needed

5. Processing Pipeline

Acquisition → Enhancement/Restoration → Morphological → Segmentation → Recognition

StageCore Function
AcquisitionSensor captures energy, digitizes. Quality ceiling for all downstream
EnhancementSubjective: make image look better. No degradation model
RestorationObjective: recover original using mathematical model of degradation
MorphologicalShape-based: extract boundaries, skeletons via structural elements
SegmentationHardest step. Partition into regions. Fails on: variability, occlusion, ambiguous boundaries
RecognitionAssign labels to segmented objects. Final classification

Mnemonic: AEMSR (Acquisition, Enhancement/Restoration, Morphological, Segmentation, Recognition)

6. System Components

  • Sensors: Convert energy → electrical signals
  • Specialized hardware: Real-time pixel-level operations
  • Computer: Runs processing algorithms
  • Software: Filters to recognition pipelines
  • Storage: 1024×1024 × 8-bit = 1 MB per image
  • Display: Visualization affects interpretation
  • Network: Remote transmission and processing
  • Hardcopy: Printed output (films, prints)

7.3 Eye vs Camera

  • Camera: Fixed focal length, moves lens to focus
  • Eye: Fixed lens-retina distance (17 mm), changes lens shape (accommodation, 14–17 mm focal length)

DIP relevance: 256 gray levels exceed cone discrimination. Resolution matches cone density. RGB maps to three cone types.

8. Image Sensing and Acquisition

Three requirements: illumination source (emits energy) + scene (reflects/absorbs/transmits) + sensor (captures, converts to signal)

Sensor Types

TypeStructureExample
SingleOne element, must scanPhotodiode
LineRow of sensors, one line at a timeFlatbed scanner
Array2D grid, full image at onceCCD/CMOS camera

Output: continuous signal → sampled (space) → quantized (intensity) → digital image.

9. Image Formation Model

f(x, y) = i(x, y) × r(x, y)

  • i(x, y): Illumination. Energy falling on point. Source-dependent. Range: 0 ≤ i < ∞
  • r(x, y): Reflectance. Fraction bounced back. Material-dependent. Range: 0 ≤ r ≤ 1
  • r = 0: total absorption (black). r = 1: total reflection (mirror)
  • Same object, different light → different image (i changes)
  • Same light, different object → different image (r changes)

X-ray case: Uses transmissivity instead of reflectance (energy passes through, not bounces).

10. Sampling and Quantization

10.1 Sampling (Space Discretization)

  • Grid M × N over continuous image. Read intensity only at grid points
  • Coordinates: x = 0, 1, ..., M−1; y = 0, 1, ..., N−1
  • More samples → finer detail. Fewer → loss of detail

10.2 Quantization (Intensity Discretization)

  • Round continuous intensity to nearest discrete level
  • Levels: L = 2k (k = bits/pixel)
  • 8-bit: 256 levels. 4-bit: 16 levels. 1-bit: binary (black/white)

10.3 Matrix and Storage

  • Stored as M × N matrix. Origin (0,0) = top-left
  • x-axis: downward (rows). y-axis: rightward (columns)
  • Storage: M × N × k bits. Example: 512×512×8 = 256 KB

11. Spatial and Intensity Resolution

AspectSpatial ResolutionIntensity Resolution
MeasuresSmallest discernible detailSmallest discernible intensity change
UnitPixels/distance (DPI)Bits/pixel (k)
Depends onSampling rate (N)Quantization levels (L = 2k)
ArtifactAliasing: jagged/blocky edgesFalse contouring: banded gradients

11.1 Iso-Preference Curves

Combinations of N (samples) and k (bits) yielding same perceived quality.

  • Shape-heavy images (text, crowds): prioritize spatial resolution (increase N). Brain fills in smooth shading
  • Shading-heavy images (portraits, surfaces): prioritize intensity resolution (increase k). Subtle gradients define form
Quick Recall
  1. Digital image: f(x,y), finite discrete, M×N pixel grid
  2. Three levels: Low (image→image), Mid (image→attributes), High (attributes→meaning)
  3. Formation: f = i × r. 0 ≤ i < ∞, 0 ≤ r ≤ 1
  4. Sampling = space discretization. Quantization = intensity discretization
  5. Storage = M × N × k bits. L = 2k levels
  6. Pipeline: AEMSR (Acquisition, Enhancement/Restoration, Morphological, Segmentation, Recognition)
  7. Segmentation is hardest: variability, occlusion, ambiguous boundaries
  8. Cones: 6-7M, fovea, color, photopic. Rods: 75-150M, periphery, dim, scotopic
  9. Eye focuses by changing lens shape (accommodation), not moving lens
  10. Spatial artifact: aliasing. Intensity artifact: false contouring
