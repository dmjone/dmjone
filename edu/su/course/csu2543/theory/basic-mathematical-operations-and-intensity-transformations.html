Slide: Neighbors of a pixel (4- and 8-neighborhood)

Concepts: A pixel's neighborhood defines adjacency and connectivity.

  • 4-neighborhood: $N_4(p)=\{(x\!-\!1,y),(x\!+\!1,y),(x,y\!-\!1),(x,y\!+\!1)\}$. 4-adjacency uses Manhattan steps.
  • 8-neighborhood: includes diagonals; $N_8(p)=\{(i,j):\max(|x-i|,|y-j|)=1\}$.
  • Use 4-adjacency for grid-like connectivity; 8-adjacency when diagonals count. Important for connected components and boundary tracing.

Slides: Paths, Connectivity, Regions, Boundaries

Concepts: How pixels form objects.

  • Path: sequence $(x_0,y_0),\dots,(x_n,y_n)$ where successive pixels are adjacent. Closed path: start=end.
  • Connected component: all pixels reachable from a pixel staying inside a set S.
  • Region: a connected set. Foreground = union of regions; background = complement.
  • Boundaries: inner boundary = pixels in region adjacent to background. Outer boundary = background pixels adjacent to region.

Slide: Distance measures

Concepts & formulas: Metrics used for distances on pixel grid.

  • Euclidean: $D_e(p,q)=\sqrt{(x-u)^2+(y-v)^2}$. (True geometric distance)
  • City-block (D₄): $D_4(p,q)=|x-u|+|y-v|$. (Grid steps)
  • Chessboard (D₈): $D_8(p,q)=\max(|x-u|,|y-v|)$. (King moves)

Slide: Element-wise vs Matrix operations

Concepts:

  • Element-wise: operate per corresponding pixel (add, multiply per-pixel).
  • Matrix product: linear algebra multiply - not per-pixel; used rarely in basic image ops.
  • Remember: arithmetic image ops require same image size.

Slide: Linear vs Non-linear operators

Concepts: Linear operator H satisfies homogeneity and additivity:

$$H[af_1 + bf_2] = aH[f_1] + bH[f_2]$$

  • Examples: mean (linear). Max, median (non-linear).
  • Implication: linear systems allow convolution representation and frequency analysis.

Slides: Arithmetic operations (add, subtract, multiply, divide)

Concepts and uses:

  • $s(x,y)=f(x,y)+g(x,y)$; $d=f-g$; $p=f\cdot g$; $v=f/g$ (element-wise).
  • Averaging multiple frames reduces zero-mean uncorrelated noise: $$\bar g(x,y)=\frac{1}{K}\sum_{i=1}^K g_i(x,y)$$ reduces variance by factor $K$.
  • Subtraction: change detection, digital subtraction angiography (remove background by subtracting pre/post contrast frames).
  • Division: shading correction when $g=f\cdot h$ so $f=g/h$ if $h$ known/estimated.

Important practical note (clipping & scaling)

Operations can produce out-of-range values (e.g., subtraction yields −255..+255 for 8-bit). Always map results back to valid range: for 8-bit, $K=255$, use clipping or linear scaling from $g_{\min}\!:\!g_{\max}$ to $0\!:\!K$.

Slides: Set & Logical operations

Set ops on images: Complement, union, intersection - apply per pixel.

  • For binary: complement = NOT, union = OR, intersection = AND, symmetric difference = XOR.
  • For grayscale arrays, union/intersection commonly implemented as per-pixel max/min (or addition capped by max value).
  • Logical ops are central in morphological/segmentation tasks.

Slides: Spatial operations (single pixel, neighborhood, geometric)

Types:

  1. Single-pixel operations: map intensity by $s=T(z)$ - e.g., negative (complement), thresholding, contrast stretching.
  2. Neighborhood operations: output at (x,y) depends on a window $S_{xy}$ around it - mean filter, median, order-statistics.
  3. Geometric transforms: change pixel coordinates (translation, rotation, scaling, shear) then interpolate intensities.

Slide: Neighborhood operations - filtering

Mean (box) filter formula:

$$g(x,y)=\frac{1}{mn}\sum_{(r,c)\in S_{xy}} f(r,c)$$

  • Mean removes Gaussian noise but blurs edges.
  • Median filter is non-linear; excellent for salt-and-pepper noise, preserves edges better.
  • Understand effect: smoothing vs edge preservation.

Slides: Geometric spatial transformations

Concepts: Affine transforms preserve lines and include rotation, scaling, translation, shear. They are matrix operations on homogeneous coordinates. Concatenate transforms by matrix multiplication:

$$A = A_n \cdot A_{n-1} \cdots A_1$$

Interpolation methods: nearest neighbor (fast, blocky), bilinear (uses 4 neighbors), bicubic (uses 16 neighbors, smoother).

Bilinear interpolation (formula & code)

For a point $(x,y)$ in between four pixels with values $f(0,0), f(1,0), f(0,1), f(1,1)$, bilinear:

$$v(x,y) = (1-a)(1-b)f(0,0) + a(1-b)f(1,0) + (1-a)b f(0,1) + ab f(1,1)$$

# Python: bilinear sample (assuming img indexed as img[y][x])
def bilinear(img, x, y):
    x0, y0 = int(floor(x)), int(floor(y))
    a, b = x - x0, y - y0
    I00 = img[y0][x0]; I10 = img[y0][x0+1]
    I01 = img[y0+1][x0]; I11 = img[y0+1][x0+1]
    return (1-a)*(1-b)*I00 + a*(1-b)*I10 + (1-a)*b*I01 + a*b*I11

Slides: Domains & Image transforms

Two domains: Spatial (pixel grid) and Transform (frequency) domain. Some filters & analyses are easier in transform domain.

2D Discrete Fourier Transform (DFT):

$$F(u,v)=\sum_{x=0}^{M-1}\sum_{y=0}^{N-1} f(x,y)\,e^{-j2\pi(\frac{ux}{M}+\frac{vy}{N})}$$

Inverse:

$$f(x,y)=\frac{1}{MN}\sum_{u=0}^{M-1}\sum_{v=0}^{N-1} F(u,v)\,e^{j2\pi(\frac{ux}{M}+\frac{vy}{N})}$$

Use Fourier to detect periodic noise (appears as bright spots) and remove it by masking in frequency domain.

Slides: Image intensities as random variables (histogram & stats)

Concepts & formulas:

  • Histogram: let $z_k$ be intensity levels, $n_k$ count of pixels with $z_k$. Probability: $$p(z_k)=\frac{n_k}{MN}.$$
  • Mean intensity: $$\mu=\sum_{k} z_k\,p(z_k).$$
  • Variance: $$\sigma^2=\sum_{k} (z_k-\mu)^2 p(z_k).$$
  • Higher central moments describe skewness/kurtosis.

Histogram equalization and contrast adjustments follow from these statistics.

# 1) Compute histogram probabilities
def hist_probs(img, L=256):
    MN = img.size
    counts = [0]*L
    for v in img.flat: counts[v]+=1
    p = [c/MN for c in counts]
    return p
