1. Color Image Fundamentals

Color image processing has two branches:

  • Full-color processing, images captured by sensors that record all color channels (e.g., a digital camera). The color is physically real.
  • Pseudocolor processing, artificial colors assigned to a grayscale image to make features easier for humans to see. The color is invented, not captured.

1.1 How Humans See Color

The human retina contains cones, photoreceptor cells sensitive to three overlapping wavelength bands corresponding roughly to Red, Green, and Blue. Every color we perceive is the brain's interpretation of the combined response of these three cone types. This is why three channels are sufficient to represent virtually any visible color.

1.2 Additive vs. Subtractive Color Mixing

There are two fundamentally different mixing systems, depending on whether you are combining light or pigments:

1.2.1 Additive Primaries (Light)

Used in monitors, projectors, cameras. You start with darkness and add light.

  • Red + Green = Yellow
  • Green + Blue = Cyan
  • Red + Blue = Magenta
  • Red + Green + Blue = White
1.2.2 Subtractive Primaries (Pigments/Inks)

Used in printing. You start with white paper and subtract light by adding ink.

  • Yellow + Magenta = Red
  • Yellow + Cyan = Green
  • Magenta + Cyan = Blue
  • Yellow + Magenta + Cyan = Black

The secondary colors of one system are the primary colors of the other. Additive secondaries (Yellow, Cyan, Magenta) are subtractive primaries, and vice versa.

2. The RGB Color Model

RGB represents colors as points in a 3D Cartesian coordinate system, a cube with Red, Green, and Blue as the three axes, each normalized to the range $[0, 1]$.

2.1 The Color Cube

  • Origin $(0,0,0)$ = Black (no light)
  • $(1,0,0)$ = pure Red; $(0,1,0)$ = pure Green; $(0,0,1)$ = pure Blue
  • $(1,1,0)$ = Yellow; $(0,1,1)$ = Cyan; $(1,0,1)$ = Magenta
  • $(1,1,1)$ = White (maximum of all channels)
  • Main diagonal from $(0,0,0)$ to $(1,1,1)$, where $R = G = B$, represents all shades of gray.

An RGB color image is stored as three independent image planes (one per channel). Each plane is a grayscale image representing that channel's intensity. The pixel at $(x,y)$ has three values: $(R, G, B)$.

RGB is the standard model for color monitors, cameras, and display hardware. Processing (enhancement, filtering) can be applied independently to each channel.

3. The HSI Color Model

RGB is hardware-friendly but does not match human intuition about color. When you describe a color as "a pale, bright orange," you are specifying three independent qualities, hue, saturation, and intensity. The HSI model separates these explicitly:

  • Hue ($H$), the dominant wavelength: what we colloquially call "the color" (red, orange, green, etc.). Represented as an angle on a color wheel (0°–360°).
  • Saturation ($S$), purity of the color. How much white is mixed in? Pure spectral color = fully saturated ($S=1$). Pastel/washed-out = low saturation. Pure gray = $S=0$.
  • Intensity ($I$), overall brightness, independent of color. Ranges from black ($I=0$) to maximum brightness ($I=1$).

Key advantage: HSI decouples intensity from color information. This means you can adjust brightness without altering the color, or modify color without changing brightness. In RGB, changing one channel affects both color and brightness simultaneously.

Practical use: Apply intensity-based operations (histogram equalization, contrast stretching) only on the $I$ channel, leaving $H$ and $S$ unchanged. This avoids color shifts that would occur if you equalized each RGB channel independently.

4. Pseudocolor Image Processing

Pseudocolor (false color) assigns arbitrary colors to gray levels in a monochrome image based on a defined mapping rule. The image was never captured in color, the color is artificially added.

4.1 Why Pseudocolor?

The human eye can distinguish only about two dozen gray levels but can differentiate thousands of color shades. By mapping gray levels to distinct colors, subtle intensity differences that are invisible in grayscale become immediately obvious in color.

4.2 Intensity Slicing

The most common pseudocolor technique. Divide the grayscale range $[0, L-1]$ into a set of intervals (slices), and assign a different color to each interval. This is a direct extension of the intensity-level slicing concept from intensity transformations, but instead of highlighting one band, you color-code the entire range.

Application: Thermal imaging (temperature maps), medical imaging (highlighting tissue density ranges), satellite imagery (land use classification).

5. Fundamentals of Image Compression

Compression reduces the amount of data needed to represent an image. An uncompressed 1024×1024 RGB image at 8 bits per channel requires $1024 \times 1024 \times 3 = 3{,}145{,}728$ bytes ($\approx 3$ MB). Compression can shrink this dramatically.

Two motivations:

  • Storage: Reduced file size means more images fit on disk.
  • Transmission: Smaller files transfer faster across networks.

Compression works by identifying and eliminating redundancy, data that is duplicated, predictable, or irrelevant.

5.1 Three Types of Data Redundancy

5.1.1 Coding Redundancy

When the binary codes used to represent pixel values are longer than necessary. Example: using a fixed 8-bit code for every intensity level, even if only 10 distinct levels actually occur in the image. Variable-length codes (shorter codes for frequent values, longer for rare ones) eliminate this redundancy.

5.1.2 Spatial (and Temporal) Redundancy

Adjacent pixels in most images are highly correlated, they have similar intensities. Encoding each pixel independently wastes bits on predictable information. Techniques like predictive coding or transform coding exploit this by encoding only the differences between pixels or the transform coefficients. Temporal redundancy extends this to video, consecutive frames are nearly identical.

5.1.3 Irrelevant Information (Psychovisual Redundancy)

The human visual system ignores certain information, fine intensity gradations, subtle color differences, high-frequency textures beyond perception. Discarding this imperceptible data is safe for human viewing but constitutes permanent information loss. This is the basis of lossy compression.

6. Fidelity Criteria: Measuring Compression Quality

Since compression (especially lossy) removes real data, we need to measure how much damage was done.

6.1 Objective Criteria

Mathematical metrics computed between the original image $f$ and the decompressed image $\hat{f}$:

  • Root-Mean-Square Error (RMSE):
$$e_{\text{rms}} = \sqrt{\frac{1}{MN} \sum_{x=0}^{M-1} \sum_{y=0}^{N-1} \left[f(x,y) - \hat{f}(x,y)\right]^2}$$

Lower RMSE = closer to original = better quality. RMSE = 0 means perfect (lossless) reconstruction.

6.2 Subjective Criteria

Human evaluators rate image quality on a scale (e.g., 1 = terrible, 5 = indistinguishable from original). This matters because RMSE does not always correlate with perceived quality, a small mathematical error in a visually important region (like a face) is worse than a larger error in a uniform background.

7. Image Compression Model

Every compression system has two halves: an encoder (compresses) and a decoder (decompresses).

7.1 Encoder: Three Stages

  • Mapper, transforms the image to reduce spatial redundancy. Example: computing pixel-to-pixel differences, or applying a transform (DCT, wavelet). The mapper is reversible, no information is lost here.
  • Quantizer, reduces psychovisual redundancy by rounding or discarding perceptually irrelevant information. This is the only lossy stage, information is permanently destroyed. Skipping this stage gives lossless compression.
  • Symbol coder, reduces coding redundancy by assigning shorter codes to more probable symbols. Example: Huffman coding. This stage is lossless.

7.2 Decoder

Reverses the encoder: symbol decoder → inverse mapper → reconstructed image $\hat{f}$. The quantizer has no inverse, quantized information cannot be recovered, which is why lossy compression is irreversible.

# Compression pipeline (conceptual)
# Encoder
mapped = transform(image)          # mapper (lossless)
quantized = quantize(mapped, step) # quantizer (LOSSY)
compressed = huffman_encode(quantized)  # symbol coder (lossless)

# Decoder
dequantized = huffman_decode(compressed)
reconstructed = inverse_transform(dequantized)

8. Huffman Coding

Huffman coding is the most widely taught lossless compression technique. It assigns variable-length binary codes to symbols based on their probabilities: frequent symbols get short codes, rare symbols get long codes.

8.1 Why It Works

A fixed-length code (e.g., 3 bits per symbol for 6 symbols) uses the same number of bits for every symbol. If symbol $a_2$ appears 40% of the time and symbol $a_5$ appears only 4% of the time, assigning 1 bit to $a_2$ and 5 bits to $a_5$ saves enormous space overall, because the short code is used far more often than the long one.

8.2 Construction Algorithm

  • Step 1: List all symbols sorted by probability in descending order.
  • Step 2: Merge the two lowest-probability symbols into a single combined node. Their combined probability is the sum.
  • Step 3: Re-sort and repeat Step 2 until only one node remains (the root).
  • Step 4: Trace back from the root to each symbol. At each branch, assign 0 to one branch and 1 to the other. The path from root to symbol gives its Huffman code.

8.3 Worked Example

Symbol Probability Huffman Code Code Length
$a_2$ 0.40 1 1 bit
$a_6$ 0.30 00 2 bits
$a_1$ 0.10 011 3 bits
$a_4$ 0.10 0100 4 bits
$a_3$ 0.06 01010 5 bits
$a_5$ 0.04 01011 5 bits

Average code length: $0.40 \times 1 + 0.30 \times 2 + 0.10 \times 3 + 0.10 \times 4 + 0.06 \times 5 + 0.04 \times 5 = 2.2$ bits/symbol. A fixed-length code for 6 symbols would require $\lceil \log_2 6 \rceil = 3$ bits/symbol. Huffman achieves 2.2, a $26.7\%$ savings.

Guarantee: Huffman coding produces the smallest possible average code length for any variable-length coding scheme where each symbol is coded independently (zero-memory source).
