1. What is a Histogram? (Core Mental Model)

An image = collection of pixels → each pixel has intensity.

A histogram answers one question:

"How many pixels have each intensity?"

1.1 Mathematical Definition

$$ h(r_k) = n_k $$

  • rₖ: intensity level
  • nₖ: number of pixels with that intensity

1.2 Normalized Histogram (Probability View)

$$ p(r_k) = \frac{n_k}{MN} $$

Interpretation: probability that a pixel has intensity rₖ

Key Insight: Histogram = probability distribution of brightness

2. Histogram = Contrast Indicator

  • Narrow histogram → low contrast (flat, dull image)
  • Wide histogram → high contrast (clear details)

Why?

Contrast ∝ spread of intensity values

Core Rule:

$$ \text{Contrast} \propto \text{Range of histogram} $$

3. Histogram Equalization (The Core Idea)

Problem: intensities clustered → poor contrast

Solution: redistribute intensities uniformly

Goal: make histogram as flat as possible

Concept: Stretch probability distribution → maximize information visibility

4. Conditions for Valid Transformation

We use: $$ s = T(r) $$

4.1 Monotonic Increasing

$$ r_1 < r_2 \Rightarrow T(r_1) \leq T(r_2) $$

Prevents: intensity reversal (artifacts)

4.2 Range Preservation

$$ 0 \leq T(r) \leq L-1 $$

4.3 Strict Monotonic (Optional)

Ensures invertibility

Concept: Preserve ordering of brightness

5. Continuous Histogram Equalization (Concept)

$$ s = T(r) = (L-1) \int_0^r p_r(w)\,dw $$

5.1 Interpretation

  • Integral = CDF (cumulative probability)
  • s = scaled CDF

5.2 Why It Works

  • CDF is always increasing → satisfies monotonicity
  • CDF(0)=0, CDF(max)=1 → correct range

Critical Insight:

Output histogram becomes uniform (ideal contrast)

6. Discrete Histogram Equalization (Actual Implementation)

$$ s_k = (L-1) \sum_{j=0}^{k} p_r(r_j) $$

6.1 Step-by-Step Algorithm

  • Compute histogram nₖ
  • Normalize → p(rₖ)
  • Compute cumulative sum (CDF)
  • Multiply by (L−1)
  • Round values
Implementation
hist, _ = np.histogram(image.flatten(), bins=L, range=[0, L])
pdf = hist / image.size
cdf = np.cumsum(pdf)
mapping = np.round((L - 1) * cdf)
output = mapping[image]

Core Insight: Equalization = apply CDF mapping

7. Key Observations (Discrete Reality)

  • Histogram not perfectly flat (rounding effect)
  • No strict proof of uniformity
  • Fully automatic (no parameters)
  • May not be reversible (many-to-one mapping)

Exam Insight: Continuous → perfect, Discrete → approximate

8. Histogram Matching (Specification)

Limitation of Equalization: Always outputs uniform histogram

Need: Control output shape

Solution: Histogram Matching

Goal: Transform image → desired histogram

9. Core Idea of Histogram Matching

Step 1: Equalize input

$$ s = T(r) $$

Step 2: Define desired distribution

$$ G(z) $$

Step 3: Map:

$$ z = G^{-1}(T(r)) $$

Concept:

Input → uniform → desired

Key Insight: Uniform distribution acts as bridge

10. Discrete Histogram Matching Algorithm

Step 1

Compute equalized values sₖ

Step 2

Compute desired CDF:

$$ G(z_q) = (L-1)\sum p_z(z_i) $$

Step 3

For each sₖ, find closest G(z_q)

Step 4

Map r → s → z

Optimization: Combine mappings directly

11. Conceptual Comparison

  • Equalization: Input → Uniform
  • Matching: Input → Custom distribution

Key Difference: Control

12. Decision Engine (Exam Gold)

  • Low contrast? → Histogram Equalization
  • Need specific look? → Histogram Matching
  • Fast automatic enhancement? → Equalization
  • Control output distribution? → Matching

Master Rule:

Equalization = maximize contrast

Matching = control appearance
