1. Decorators Overview

A decorator is a function whose primary purpose is to wrap another function or class to transparently alter or enhance its behavior. They are denoted using the special @ symbol, providing a clear and concise way to apply additional functionality.

1.1 How Decorators Work

At its core, a decorator takes in a function, adds some functionality to it, and returns it. This process is facilitated by the @ symbol placed above the definition of the function that is to be decorated. The effect is the automatic application of a higher-order function that modifies or enhances the original function or method without changing its source code.

@decorator
def some_function():
    # Function body

This is equivalent to:

def some_function():
    # Function body
some_function = decorator(some_function)

1.2 Creating Decorators

To create a decorator, you define a function that takes another function as its argument. Within this outer function, you define a nested function that wraps the functionality of the original function. The outer function then returns this nested function.

def my_decorator(func):
    def wrapper():
        # Code before function call
        func()
        # Code after function call
    return wrapper

1.3 Using Decorators

Decorators can be applied to functions and methods to modify their behavior at call time. This allows for logging, access control, memoization, and other patterns to be easily implemented.

@my_decorator
def my_function():
    print("Hello World")

This applies my_decorator to my_function, modifying its behavior without altering its internal code.

1.3.1 Practical Example: Logging

A common use case for decorators is logging function calls, which can help with debugging and monitoring. Here’s how a simple logging decorator might look:

def logger(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__} with {args} and {kwargs}")
        result = func(*args, **kwargs)
        print(f"{func.__name__} returned {result}")
        return result
    return wrapper

This decorator logs the function name, its arguments, and its return value each time the decorated function is called.

1.4 Decorators with Arguments

Some decorators may need to accept arguments in addition to the function they decorate. This can be achieved by adding another level of function nesting:

def repeat(num_times):
    def decorator_repeat(func):
        def wrapper(*args, **kwargs):
            for _ in range(num_times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator_repeat

This allows for a decorator to be used as @repeat(3), indicating the decorated function should be repeated three times.
