Very Short Definitions

  1. Goal of Computer Forensics

    • Reconstruct “what happened” on a digital system
    • Identify who did it, preserve evidence for court

  2. Transnational Crime

    • Illegal acts crossing national borders (e.g., hacking servers in another country)

  3. Identity Fraud

    • Stealing someone’s personal data (SSN, bank info) to impersonate them

  4. Phishing Attack

    • Fake emails/websites tricking you into giving passwords or credit-card data

  5. Hacking

    • Unauthorized intrusion into or control over someone else’s computer or network

Short-Answer Topics

1. Types of Forensics Technology

  • Disk Imaging (bit-for-bit “clone” of drives)

  • Network Forensics (log-file & packet analysis)

  • Mobile/Cloud Forensics (smartphones, VMs)

  • Live vs. Static

    • Static: machine is off; image whole disk
    • Live: machine is on; capture RAM + volatile data

2. Cybercrime Basics

  • Categories

    • Against persons: harassment, stalking, doxing
    • Against property: malware, ransomware, piracy
    • Against government: cyber-terrorism, infrastructure attacks

  • Examples

    • Hacking (break-in)
    • Phishing (credential theft)
    • Ransomware (data-locking extortion)
    • Cyber squatting (selling famous domains)

3. Cyber Stalking vs. Cyber Squatting

Aspect Cyber Stalking Cyber Squatting
What it is Repeated harassment or threats online Registering famous domains to resell at profit
Victim harm Psychological distress, fear Economic loss, brand damage
Key clue threatening messages, tracking user domain parked, “for sale” notices

Long-Answer Outlines

1. Users of Forensic Techniques

  • Law Enforcement: build criminal cases
  • Corporate/HR: internal fraud, policy violations
  • Civil Litigators: e-discovery in lawsuits
  • Insurance: fraud investigations
  • Individual Experts: private clients

2. Password-Cracking Methods

  • Brute-Force: try every combination (guaranteed, slow)
  • Dictionary Attacks: use wordlists (fast vs. common)
  • Rainbow Tables: precomputed hashes → instant lookup
  • GPU/Cloud Cracking: parallelize for speed

3. Primary Objectives & Challenges (Term 2)

  • Objectives:

    • Identify, preserve, analyze, document, present digital evidence

  • Challenges:

    • Encryption & hidden data
    • Volatile evidence (RAM)
    • Rapidly evolving tech & formats
    • Legal admissibility

4. Chain of Custody

  1. Label & bag evidence on-scene
  2. Log every hand-off (who, when, why)
  3. Use write-blockers for drives
  4. Hash before/after imaging (MD5, SHA-1)
  5. Store in locked, access-controlled evidence locker

5. Five Phases of Forensics Process

  1. Identification: spot devices/logs with potential evidence
  2. Preservation: isolate & protect evidence; maintain CoC
  3. Collection: acquire images (static/live) & network captures
  4. Examination/Analysis: recover deleted files, parse logs, metadata
  5. Presentation: report findings; expert testimony

6. Investigator’s Role

  • Plan & scope investigation
  • Select tools & techniques
  • Execute imaging & analysis
  • Write clear, factual reports
  • Testify in court as subject-matter expert

7. Types of Digital Evidence

  • Disk Images (files, folders, slack space)
  • Memory Dumps (RAM: passwords, keys)
  • Logs (web, system, application)
  • Network Captures (PCAPs, proxy logs)
  • Browser Artifacts (history, cookies)

8. Legal & Ethical Considerations

  • Privacy Laws: only examine authorized data
  • Admissibility: follow standards (Daubert, Frye)
  • Affidavits & Warrants: obtain search warrants before seizure
  • Impartiality: remain unbiased; document every step

9. Data Acquisition: Live vs Static

Aspect Static Acquisition Live Acquisition
Machine state Powered off (“dead”) Powered on; capture RAM & processes
What you get Full disk image Disk + volatile data (RAM, handles)
Risk No changes, but miss live data Risk of altering system; more evidence types

10. Cybercrime Types & Forensic Value

  • Malware/Ransomware: file-time analysis, C2 server logs
  • Phishing: mail headers, site-forensics, victim PC images
  • Insider Threats: access logs, privilege escalation traces
  • Network Attacks: packet captures, firewall logs

11. Common Forensic Tools

  • Disk Imaging: dd, EnCase (.E01), FTK Imager
  • Password Cracking: Hashcat, John the Ripper
  • Memory Analysis: Volatility, Rekall
  • Network Forensics: Wireshark, tcpdump
  • Log Analysis: LogParser, Splunk

Ready-to-Write Bullets

A. NIST Five Phases of Forensic Process

  1. Identification – Spot devices, logs, VMs with evidence
  2. Preservation – Isolate systems; use write-blockers; hash drives
  3. Collection – Acquire disk image (static) or RAM + disk (live)
  4. Examination/Analysis – Recover deleted files; parse metadata; log correlation
  5. Presentation – Document steps; generate report; testify in court

B. Chain of Custody

  • Purpose: Prove evidence integrity from seizure to courtroom
  • Key Steps:
  1. Label & Seal: Unique ID on each item
  2. Log Entries: Who-when-why each transfer
  3. Hashing: MD5/SHA1 before & after imaging
  4. Secure Storage: Locked evidence locker
  • Court Impact: Any gap → evidence inadmissible

C. Types of Digital Evidence

Evidence Type What It Is Why It Matters
Disk Images Bit-for-bit copy of storage drives Recover files, slack/unallocated
Memory Dumps Snapshot of RAM Passwords, encryption keys, processes
Logs System, web-server, application logs Timeline of events, user actions
Network Captures PCAPs or firewall logs Source/destination, payload content
Browser Artifacts History, cookies, cache User behavior, site visits

D. Live vs. Static Acquisition

Aspect Static (Dead) Acquisition Live Acquisition
State System powered off System running
Data Captured Full disk only Disk + volatile RAM, processes
Risk No alteration, but misses live Alters system; captures more types
Use Case Malware-free image creation Incident response; decrypt RAM-held keys

E. Password-Cracking Methods

  1. Brute-Force – All combos → guaranteed but slow
  2. Dictionary – Common words/phrases → fast for weak passwords
  3. Rainbow Tables – Precomputed hash-lookup → instant if table exists
  4. GPU/Cloud Cracking – Parallelize for speed (Hashcat, JtR)

N. File Signatures & Extension Mismatches

  • File Signature (Magic Number): First few bytes (e.g. FF D8 FF for JPEG) that uniquely identify a file’s true type
  • Extension Mismatch: When filename extension (e.g. .jpg) doesn’t match signature (e.g. PDF magic); red flag for tampering

O. Forensic Soundness & SOP Elements

  • Forensic Soundness: Evidence collected without alteration, reproducible steps, full transparency
  • Key SOP Elements:
  1. Tool Validation – verify imaging/analysis tools
  2. Evidence Handling – write-blockers, sealed bags
  3. Documentation – step-by-step logs
  4. Chain of Custody – record transfers (who/when/why)
  5. Analyst Competence – trained personnel

P. NIST Incident Response Lifecycle (6 Phases)

  1. Preparation – policies, training, tools in place
  2. Identification – detect & declare incident
  3. Containment – short-term & long-term control
  4. Eradication – remove root cause (malware, vuln)
  5. Recovery – restore systems to production
  6. Lessons Learned – post-mortem & update plans

Q. Virtualization Forensics Essentials

  • Artifacts:

    • VM Snapshots: point-in-time disk + memory
    • Hypervisor Logs: host events (VM start/stop)
    • Virtual Disk Files (e.g. .vmdk): file system data

  • Challenge: Layered snapshots hide changes

  • Mitigation: Preserve all snapshot levels & metadata

R. Key Logs & Evidence Locker

  • Syslog: system events, auth attempts, kernel messages
  • Web-Server Access Logs: client IP, URL, timestamps, status codes
  • Evidence Locker: locked, access-controlled storage; digital hashes tied to physical tags

S. Frye vs. Daubert Standards

  • Frye: Evidence methods must be “generally accepted” by the relevant scientific community
  • Daubert: Judge evaluates admissibility—factors include testability, peer review, known error rate, standards

T. Common Disk‐Image Formats

  • Raw (dd): bit‐for‐bit, no metadata
  • E01 (EnCase): includes case info, checksums, compression
  • AFF: open standard, metadata + compression

U. Web‐Server Logs: Indicators of Compromise

  • Access Logs (IP, timestamp, request, status)
  • Error Logs (application errors, stack traces)
  • Top IoCs:
  1. Burst of requests (DDoS or brute-force)
  2. Repeated 404/403 to hidden endpoints
  3. Suspicious POST payloads (SQLi, XSS signatures)

V. Correlating Logs Across Sources

  • Timeline Stitching: align events by timestamp (use NTP-synced logs)
  • IP Cross-Reference: map malicious IPs between firewall, web, network captures
  • Event Linking: match a user’s failed login (auth log) to subsequent file access (app log)

W. Hypervisor & Syslog

  • Hypervisor: Software layer hosting and managing virtual machines
  • Syslog: Standard protocol/daemon for recording system and application messages

Y. Acquisition & Verification Essentials

  1. Write-Blocker: prevent changes to original media
  2. Hashing: compute MD5/SHA-1 before and after imaging
  3. Tool Validation: confirm imaging tools produce bit-identical copies
