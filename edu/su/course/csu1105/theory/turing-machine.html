Turing Machine

1. Turing Machine Overview

A Turing Machine is a theoretical machine that manipulates symbols on a strip of tape according to a table of rules. Despite its simplicity, the model is powerful enough to simulate the logic of any computer algorithm, and is thus used as a standard model of computation for determining the limits of what can be computed.

1.1 Components

The Turing Machine is composed of:

  • Tape: An infinite length of tape divided into contiguous cells. Each cell contains a symbol from some finite alphabet. The tape is bidirectional, allowing the machine to move left and right.
  • Head: The head reads and writes symbols on the tape and can move to the left, right, or stay in place after reading a cell.
  • State register: Holds the state of the Turing machine. The number of states is finite.
  • Table of rules (transition function): Dictates the behavior of the machine. For each pair of current state and symbol read from the tape, it specifies (1) the symbol to write, (2) the direction to move the head (left, right, or stay), and (3) the subsequent state.

1.2 Formal Definition

A Turing Machine can be formally defined as a 7-tuple (Q, Σ, Γ, δ, q0, qaccept, qreject):

  • Q: Finite set of states
  • Σ: Input alphabet not containing the blank symbol (usually represented as 'Β' or '_')
  • Γ: Tape alphabet, where Σ ⊆ Γ and Γ includes the blank symbol
  • δ: Transition function mapping Q × Γ to Q × Γ × {L, R, S} (Left, Right, Stay)
  • q0: Initial state
  • qaccept: Accept state
  • qreject: Reject state, where qreject ≠ qaccept

1.3 Operation

The Turing Machine begins with the tape head positioned on the leftmost symbol of the tape and in the initial state. The machine operates according to the rules defined in δ, writing symbols, moving the head, and changing states. Computation continues until the machine reaches either the accept or reject state, at which point it halts.

1.4 Example

Consider a simple Turing Machine designed to recognize strings containing an equal number of '0's and '1's.

Initial state: q0
Input tape: '0011'
Transition rules:
(q0, 0) → (q1, X, R)  // Cross off a '0', move right
(q1, 0) → (q1, 0, R)  // Skip '0's
(q1, 1) → (q2, X, L)  // Cross off a '1', move left
(q2, X) → (q2, X, L)  // Move left to find uncrossed '0'
(q2, 0) → (q0, X, R)  // Cross off a '0', repeat
...
(q0, _) → (q_accept, _, S)  // If empty, accept

This machine uses 'X' to mark off processed symbols and moves between states to track the process until it verifies that the number of '0's equals the number of '1's, or it reaches an inconsistency and rejects.
