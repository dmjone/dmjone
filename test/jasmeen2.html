Part 1: Introduction to Problem Decomposition (30 minutes)

1.1 Understanding Problem Decomposition

Definition: Breaking down a complex problem into smaller, more manageable parts.

Importance: Simplifies problem-solving, makes debugging easier, and enhances understanding.

1.2 Steps in Problem Decomposition

  • Identify the Main Problem: Understand the overall goal or problem statement.
  • Break Down into Sub-Problems: Divide the main problem into smaller, logical sub-problems.
  • Solve Each Sub-Problem: Focus on solving each sub-problem independently.
  • Integrate Solutions: Combine the solutions of sub-problems to solve the main problem.

1.3 Example of Problem Decomposition

Example: Planning a Party

  • Main Problem: Organize a successful party.
  • Sub-Problems:
    • Decide on a date and time.
    • Create a guest list.
    • Send invitations.
    • Plan the menu.
    • Arrange for entertainment.
    • Set up the venue.

Part 2: Introduction to Algorithms (30 minutes)

2.1 What is an Algorithm?

Definition: A step-by-step procedure to solve a problem or perform a task.

Importance: Provides a clear and structured approach to problem-solving.

2.2 Characteristics of a Good Algorithm

  • Finiteness: An algorithm must terminate after a finite number of steps.
  • Definiteness: Each step must be precisely defined.
  • Input: An algorithm should have zero or more inputs.
  • Output: An algorithm should produce at least one output.
  • Effectiveness: Each step must be simple enough to be carried out.

2.3 Example of an Algorithm

Example: Making a Sandwich

  • Step 1: Gather ingredients (bread, butter, cheese, etc.).
  • Step 2: Spread butter on one side of a slice of bread.
  • Step 3: Place a slice of cheese on the buttered side.
  • Step 4: Cover with another slice of bread.
  • Step 5: Cut the sandwich into halves.

Definition: A high-level description of an algorithm using plain language and structured format.

3.4 Writing Pseudocode

Algorithm to calculate the factorial of a number:

  • Start
  • Input number

  • Output factorial
  • End

4.2 Design Algorithms

Activity: Simple Algorithms

Problem 1: Create an algorithm to find the largest of three numbers.

  • Pseudocode:
  • Start
  • Input three numbers
  • If the first number is greater than the second and third numbers, it is the largest.
  • Else if the second number is greater than the first and third numbers, it is the largest.
  • Else, the third number is the largest.
  • Output the largest number
  • End

# Algorithm to find the largest of three numbers

def find_largest(num1, num2, num3):
    if num1 > num2 and num1 > num3:
        largest = num1
    elif num2 > num1 and num2 > num3:
        largest = num2
    else:
        largest = num3
    return largest

# Example usage
num1, num2, num3 = 10, 25, 15
print("The largest number is:", find_largest(num1, num2, num3))

Problem 2: Create an algorithm to check if a number is even or odd.

  • Pseudocode:
  • Start
  • Input a number
  • If the number modulo 2 is zero, it is even.
  • Else, it is odd.
  • Output whether the number is even or odd
  • End


# Algorithm to check if a number is even or odd

def check_even_odd(number):
    if number % 2 == 0:
        return "Even"
    else:
        return "Odd"

# Example usage
number = 42
print("The number is:", check_even_odd(number))

  • End

  • End
