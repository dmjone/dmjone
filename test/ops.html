  • Seals a PDF with a CAdES‑detached signature (shown as PAdES in PDF readers).
  • Adds DSS (certs/CRLs/optional OCSP) so it can be validated offline (PAdES B‑LT).

How it works

Step 1

Upload

Admin uploads PDF (or API). Worker calculates SHA‑256 and records an audit seed.

Step 2

Seal

Java signer (PDFBox 3.0.6 + BouncyCastle 1.82) applies a CAdES signature and embeds DSS (certs/CRLs/OCSP).

Step 3

Timestamp

Optional RFC 3161 timestamp for B‑LTA longevity, via our TSA.

Step 4

Verify

Anyone can verify online. PDF readers validate offline. Revocations take effect immediately.

Stack (from your code): Java 21 (Javalin 6.7.0), Apache PDFBox 3.0.6, BouncyCastle 1.82, OpenSSL CA/OCSP/TSA (RFC 3161), Cloudflare Worker + D1, NGINX reverse proxy.

FAQ

Is this an e‑signature platform like DocuSign?
No. This is a document sealing & verification system. It signs documents with your organization’s certificate and lets anyone verify authenticity. If you need multi‑party workflows, templates, or bulk signer routing, use an e‑signature platform; we price accordingly lower when those features are out of scope.
Will my PDFs show as trusted by default in Adobe?
By design this uses a private trust model (your Root/ICA). That means recipients should install your Trust Kit to get default green checks. If you require public trust (AATL/EUTL), that’s a different product & price point.
