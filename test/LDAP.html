LEVEL 1: What is LDAP? (Layman Terms)

LDAP = Lightweight Directory Access Protocol.

Think of it like a phonebook for computers.

It stores and retrieves data about people, systems, permissions, etc., in a hierarchical format.

LEVEL 2: Why LDAP? (Real-World Usage)

Used to centralize user information: login credentials, email, group memberships, roles, etc.

Popular in corporate IT environments to manage access to services like email, VPN, shared drives.

Replaces the need to manage separate logins for every service (SSO-style).

LEVEL 3: How LDAP Works (Core Concepts)

  • Directory: A hierarchical database optimized for read-heavy access. Structured like a tree with nodes and leaves.
  • Entry: Basic unit of data in LDAP. Example: a user, group, or printer.
  • DN (Distinguished Name): Like a full address/path of an entry.
    Example: cn=John Doe,ou=Employees,dc=example,dc=com
  • RDN (Relative Distinguished Name): Just the last component of the DN.
    Example: cn=John Doe
  • Attributes: Key-value pairs inside an entry.
    Example:
    cn: John Doe
    mail: john.doe@example.com
    uid: jdoe
  • Schema: Defines what object types (users, orgs, etc.) and what attributes are allowed.

LEVEL 4: LDAP Structure (Hierarchy)

dc=example,dc=com          ← Root
 ├── ou=People             ← Organizational Unit
 │    └── cn=Alice         ← Entry (User)
 └── ou=Groups             ← Organizational Unit
      └── cn=Admins        ← Entry (Group)
  • dc = domain component
  • ou = organizational unit
  • cn = common name

LEVEL 5: Protocol Details

Runs over TCP/IP (usually port 389, or 636 for LDAPS)

Uses operations like:

  • bind: authentication
  • search: query
  • add: add entry
  • modify: update entry
  • delete: remove entry
  • unbind: logout

LEVEL 6: Authentication & Authorization

Bind operation is used for authentication.

LDAP servers can be used for:

  • Authentication: Who are you?
  • Authorization: What are you allowed to do?

Common LDAP-compatible auth systems:

  • Active Directory (AD) – Microsoft's LDAP + Kerberos-based service
  • OpenLDAP – Free and open-source LDAP implementation
  • ApacheDS, Novell eDirectory, etc.

LEVEL 7: Integration Use Cases

LDAP is used by:

  • SSO systems (Login to Google, GitLab, etc.)
  • Email servers (Postfix, Dovecot)
  • Operating Systems (Linux PAM modules, Samba)
  • Web Apps (using LDAP plugins or SAML/SSO modules)

LEVEL 8: Real-World Questions You Might Be Asked

  1. How is LDAP different from a database like MySQL?
    LDAP is optimized for reads, hierarchical, and schema-restricted.
    DBs are relational, flexible, and support complex queries and joins.
  2. What’s the difference between LDAP and Active Directory?
    AD = Microsoft’s implementation of LDAP + Kerberos + GPO + extra services.
    LDAP is the protocol; AD is a directory service that implements it.
  3. How do you secure LDAP?
    Use LDAPS (LDAP over SSL/TLS).
    Use simple bind over TLS or SASL-based authentication methods.
  4. Can LDAP be used for group-based access control?
    Yes. Define groups (entries), and add user DNs to group member attributes.

Optional Tools

  • ldapsearch – CLI tool to search LDAP directories.
  • Apache Directory Studio – GUI to view and manage LDAP directories.
  • ldif – LDAP Data Interchange Format, used to import/export data.

Sample LDAP Search Command (Linux CLI)

ldapsearch -x -H ldap://ldap.example.com -D "cn=admin,dc=example,dc=com" -W -b "dc=example,dc=com" "(uid=jdoe)"
